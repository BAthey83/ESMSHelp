Functionality Index and Building Functionality Index

This topic defines and discusses the BUILDER terms "Functionality Index" and "Building Functionality Index ."

Functionality Index

The Functionality Index (FI) is a performance based index that indicates losses in building performance from general obsolescence, new codes/standards, or updated user requirements. Functionality loss is a function of (1) functionality sub-issue type, (2) sub-issue severity, and (3) sub-issue density.

The FI helps determine to what degree a building's size, location, and configuration support its designated purpose, as well as the safety and well-being of its occupants.

  • the second Ʃ represents the number of individual severity levels present for ith issue,
  • a() represents the functionality deduct value for sub-issue type (I), severity (S), and density (D),
  • and F() is the adjustment factor when multiple sub-issues are present based on total summed deduct (t), and number of sub-issues present (d).

The Functionality Index is one of two indexes used to calculate another index called the Performance Index (the other contributing index is the Condition Index). The Performance Index is what determines the color (green, amber, or red) shown for an inventory asset.

In a nutshell, BUILDER calculates the Performance Index by first collecting the Condition Index value and the Functionality Index value applicable to the target inventory asset. Next, the lower index value is multiplied by 60 percent and the higher index value is multiplied by 40 percent. Finally, the two numbers are added together to determine the Performance Index value for that inventory asset.

Note: Until a functionality assessment is performed on a Building, the Functionality Index for that Building is 100 percent by default.

Building Functionality Index

The Building Functionality Index (BFI) measures the functionality of the Building as a whole. It is computed from the results of a functionality assessment of the Building as a whole.

All FI metrics are based on a scale of 0-100, which matches the scale used in BUILDER for condition indices (BCI, SCI, BCCI, and CSCI). BUILDER contains the model algorithms that compute the FIs automatically once the functionality assessment data are entered. The algorithms use the data entered regarding the functionality issues present in the Building, the severity of the issues, and the density of each issue to calculate the BFI. It is important to note that the FI metrics are step functions over time, unlike the smooth, continuous functions seen in BUILDER's condition indices. This is because functionality, unlike condition, is not an ongoing process.

The functionality of a Building or of a Functional Area depends on user needs, technical obsolescence, and compliance with code or regulations.
