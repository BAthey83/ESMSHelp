Synchronize Enterprise SMS

This topic explains how to synchronize Enterprise SMS between your Web version and the Enterprise SMS field application.

As illustrated in the diagram below, Enterprise SMS Web and the Enterprise SMS field application are synchronized with each other through a mutual server.

Enterprise SMS Web is automatically synchronized. Enterprise SMS Web automatically updates the server with each change you make, and changes received by the server from Enterprise SMS Desktop or Tablet are automatically pushed to your Enterprise SMS Web instance.

WARNING: If you work with more than one site, note that these automatic updates happen only for the site that is designated as your synch site.

The Enterprise SMS field application must be manually synchronized. In the field application, Enterprise SMS Desktop and Enterprise SMS Tablet must each be manually synchronized to the server. $$query m2 about startup

 

Select the Field Application Synch Site in Enterprise SMS Web

If you work with more than one site, you can use the "Application Synch Settings" option in the Enterprise SMS Web user menu (at the far right on the menu bar) to change which site will appear when you use the field application.

 

[Synchronize Enterprise SMS Web with All Permitted Sites]

If your instance of Enterprise SMS Web lists multiple permitted sites, extra steps are needed to synchronize all of them:

  1. Make a list of all permitted sites, starting with the one that is currently your designated synch site. $$Revisit app1 to see how this is designated.
  2. For each permitted site from the second one in the list to the last one, do the following steps:
  3. Open the user menu.
  4. Select "Application Synch Settings".
  5. In the popup, select the site name.
  6. Activate the Set Sych Target button.
