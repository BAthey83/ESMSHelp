ESMS Access Control Permissions Management

This topic details the hierarchy of roles and permissions for users and administrators of Enterprise SMS.

WARNING: The privileges assigned to each user Role may evolve over time. Although the capabilities listed below are generally correct, they may not fully correspond to each Role at a given time.

Introduction to User Permissions

Enterprise SMS is designed to suit the needs of a wide range of users. As a result, all users are required to be assigned to a team. Each team contains different permission levels that correspond to each Organization, Site, or Group. These permission levels allow each user to only have the access they need to carry out their duties.

This flow chart details how users progress from the Account Creation process to obtaining their requested Permissions.

This topic demonstrates how to obtain Administrator permissions, how to set up Team permissions, and how to assign users to Teams.

Permissions Scopes

Each user is assigned to one or more Roles. Roles are allowed varying degrees of Permission Scopes within Enterprise SMS.

Permission Scopes include:

  • Global: Grants access to all data from Organizations, Users, and Catalogs.
  • Organization: Grants access to all Organization level data, including all Sites and Groups within that Organization.
  • Site: Grants access to Site level data, including all Groups within the Site.
  • Group: Grants access to data related to all of the Facilities within a specified Group.

Permission Definitions

Note: Enterprise SMS Permissions are structured within a strict hierarchy. For additional details on how Permissions are inherited within the structure, refer to the Custom Role Permissions table below.

  • Security Administrator: The highest level of Administrator, this role provides global permissions across the entirety of Enterprise SMS.
  • Assessment Supervisor: This role gives the user Add/Edit/Delete permissions for Assessments, as well as Lock/Unlock privileges.
  • Organization Administrator: This privileged role can elevate the privileges of other users within an Organization, including system data rights. No Security Administrator privileges are allowed.
  • Organization Curator: Users can update Reference Books and Funding for an Organization. Access to the Permissions tab is not allowed.
  • Site Administrator: This privileged role can elevate the privileges of other users in a Site, including system data rights. No Security Administrator privileges are available.
  • Site Curator: This role allows users to update Reference Books and Funding for a Site. Access to the Permissions tab is not allowed.
  • Inventory Read: This role gives users the ability to view all the available data within the Inventory tab.
  • Inventory Create: This role gives users the ability to Create Components and documents within the Inventory tab.
  • Inventory Update: This role allows users to make updates to Component data, as well as add images to Assets and Components.
  • Inventory Delete: The Inventory Delete role lets users remove Components and Documents from the Inventory tab.
  • Assets Create: This role lets users make new Assets within Enterprise SMS.
  • Assets Update: This role lets users make revisions to existing Asset data.
  • Assets Delete: This role lets users remove Assets from Enterprise SMS.
  • Assessor: Enterprise SMS Assessors can make new Assessements. In addition, Assessors can edit Assessments as long as they are owned by the Assessor, are within a preconfigured time limit, and are unlocked.
  • Work Planner: The Work Planner role can Create/Edit/Delete Work Items, Work Plans, and Projects. Additionally, Work Planners can perform Work Analysis as needed.
