Add Scheme

Create and Name the Scheme

To add a scheme,

  1. On the Workflow editor menu bar, activate the New Scheme button. The "Create New Scheme" popup will appear.

    WARNING: If you skip this step and just start dragging state boxes to the work area, you will not be able to save the scheme you build!

  2. Enter desired scheme name in the Name field.
  3. Activate Create. (Alternatively, activate the Close button to not add the scheme.)

As soon as you have created a new scheme, you should see (1) a left-hand column with some boxes, and (2) a wider blank space to the right, which is the work area where you will place elements into a work flow diagram.

Add States and Transitions

About States and Transitions

The elements of a diagram are states and transitions, as defined below:

A scheme is comprised of states and transitions:

  • A state represents the condition that the process is in at a specific time.
  • A transition defines how the process will move from one state to the next.

A scheme may also make use of parameters to store variables that are global to the scheme.

In the Workflow editor, a scheme's states and transitions are presented as a diagram. The states appear as boxes in the diagram. The transitions are represented by lines connecting the boxes. For clarity, each state should be named; transition names are optional.

Add a State to the Diagram

To add a state to the diagram (after properly creating and naming the diagram following the instructions in the previous section), drag one of the state boxes from the left-hand pane into the work area:

  • The green box represents the state that is the first state in the work flow.
  • A gray box represents a middle state.
  • The red box labeled "Finish" represents the state that is the last state in the work flow. To display as red, this box must be dragged into the work area. If a state is simply re-labeled as "Final," it will not change color in the diagram.

To view or edit details about a state, activate the state box in the diagram and the corresponding "State" popup will appear with the details.

State details are fairly simple:

  • Name - Name of the state, which appears in the state box on the diagram. This can be changed.
  • Initial / Final - If the state is the first state in the work flow, the Initial checkbox should be marked; if the state is the last state in the work flow, the Final checkbox should be marked (this information is also indicated by colors in the diagram).
  • Code Action - The "Reset all approvals" code action allows an approval to be re-used for a subsequent transition without having to rename it. A "set X stage" code action creates a linking between the state and an assessment stage. Code actions are executed when the state is entered.

Note: Only the Close button will show in the popup. Whatever the data is in the popup at the time you close it is what will be saved.

Add a Transition to the Diagram

Once you've added at least two state boxes to the diagram area, you can add a transition between any two state boxes as follows:

  1. Hover at the edge of the first state box until a pointing hand appears as the mouse position indicator.
  2. Hold down the mouse button and drag from the originating state box towards the next one in the work flow sequence to create a transition, which will appear as a connecting line. When the two connected boxes appear with colored outlines around them, you know that the Workflow editor has recognized the connection that will be made when you release the mouse button.
  3. Release the mouse button.
  4. To start specifying properties for the transition, double-click on the line to open the "Transition" popup.
  5. Name (Optional) - Enter a name for the transition.
  6. From State (Required) - Select from dropdown.
  7. To State (Required) - Select from dropdown.
  8. Type (Required) - Select from dropdown.
  9. Additional Attributes - Additional attributes are described in the topic Edit Scheme Transitions.
  10. Close the transition popup.

Note: Only a Close button will show in the popup. Whatever the data is in the popup at the time you close it is what will be saved.

  1. IMPORTANT: If you are finished or close to finished with designing the Workflow diagram, be sure to activate the Save icon button on the Workflow editor menu bar. The diagram will NOT automatically save when you exit.

You can see more about transition details at Edit Scheme Transitions.

Add a Self-Referencing Transition to the Diagram

You can add a "transition" that doesn't actually progress from one state to another, but returns to its originating state. In this type of transition, the "From" and "To" states are the same.

Why a self-referencing transition? A self-referencing transition can be a useful tool for accumulating multiple approvals or other conditions that all need to be met before progressing to the next state.

To construct a self-referencing transition on the diagram,

  1. Hover at the edge of the state box until a pointing hand appears as the mouse position indicator.
  2. Hold down the mouse button and drag outward from the box.
  3. When a colored outline appears around the box, drag back to the box and release the mouse button. The "transition" should appear in the form of a roughly semicircular outline with an arrow at one end.
  4. To start specifying properties for the transition, double-click on the line to open the "Transition" popup.
  5. Name - Enter a name for the transition.
  6. From State - Select from dropdown.
  7. To State - Select the same state again. This action is what makes it a self-referencing transition.
  8. Type - Select from dropdown.

You can see more about transition details at Edit Scheme Transitions.

Delete a State or Transition from the Diagram

To delete a state from the diagram, see instructions in the topic Edit Scheme States.

To delete a transition from the diagram, see instructions in the topic Edit Scheme Transitions.

Save and Exit

Diagram creation does not auto-save. (This allows more freedom of creative experimentation.) When you are at a design point that you are happy with, and again before exiting, make sure to activate the Save icon button on the Workflow editor menu bar.

To exit the Workflow editor, return to the Dashboard via the Interagency ESMS logo.
