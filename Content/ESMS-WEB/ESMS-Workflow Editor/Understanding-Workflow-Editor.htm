Understanding the Workflow Editor

This topic introduces tasks you can perform at the Workflow Editor screen and provides links to greater detail for each.

When you use Workflow in conjunction with an assessment, it is the scheme that defines the work flow process, the steps to be followed in creating an assessment. Each scheme is created by someone in the user's organization who has appropriate permissions to do so. If the Workflows tile ( ) does not appear on your menu, you do not have permissions.

What Can I Do Here?

The Workflow editor screen is where you can view, add or edit workflow schemes to be used with Assessment Plans.

Permissions Note : Only users with "WorkflowRole" permission set at an ESMS Administrator's Security screen can access the Workflow editor screen.

CAUTION: Because the workflow scheme design process is very open and flexible, "WorkflowRole" permission should be given out sparingly, to those who can anticipate the consequences of their design decisions.

If your workflow needs to follow a scheme that does not yet exist, you can use the Workflow editor to create a new scheme. If you need to edit a scheme for it to appropriately guide the workflow of a given assessment, take careful note of the message below.

IMPORTANT: If you need to edit a scheme in order for it to appropriately guide the workflow of a given assessment, make sure to do all edits necessary BEFORE assigning the scheme to the assessment at the "Create Assessment" popup. The assessment will function from a snapshot of the scheme taken at the time of creating the assessment, and will not reflect any changes made later to the scheme.

Workflow Menu Bar

When you enter the Workflow editor, three options are available on the Workflow menu bar:

  • Open Scheme - Activate to open an existing scheme by selecting it from the dropdown list in the "Open Scheme" popup. To open a scheme,
    1. Activate Open Scheme on the Workflow menu bar. The "Open Scheme" popup will appear.
    2. In the popup, activate the down arrow in the blank field to show the list of existing schemes.
    3. Select a scheme from the list.
    Note: The selected scheme will appear in the work area immediately, but your choice isn't fixed until you activate Ok.
    1. If you wish, you can change your choice at this point by re-activating the down arrow and making a different selection.
    2. Activate Ok.
    3. From here, you can view the scheme or edit its states and/or transitions. To read more about states and transitions, see the topic Add Scheme.
  • Create New Scheme - Activate to add a scheme. See the topic Add Scheme for more information.
  • Workflow Permissions - Activate to manage Workflow permissions. Workflow permissions are at a higher level than individual schemes. Permissions set up here can be used in any one of the schemes. When a particular permissions role is called for in a scheme, then in any assessment using that scheme, one or more individuals will need to be assigned to that Workflow permissions role when the assessment is being set up. To learn more about Workflow permissions, see the topic Manage Workflow Permissions.

Note: Workflow permissions are not the same thing as general ESMS permissions for Teams or individuals.

Once a new or existing scheme is open in the work area, additional icon buttons become available on the menu bar:

  • Save - Activate to save the scheme displayed in the work area.
  • Scheme Parameters - Activate to manage scheme parameters, which are variables global to the scheme. For more information, read Edit Scheme Transitions and Manage Scheme Parameters.
  • Scheme Details - Activate to view scheme details. (Currently, this contains only the scheme name.)
