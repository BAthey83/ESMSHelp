Permissions Needed for Assessments/Workflow

  • To edit or view information in stages of an assessment after it has been created, you need the ESMS permission named "Assessment Editor.". This can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission, to edit or view any Assessment.
  • To create an assessment, one either with or without workflow, you need "AssessmentRole" permission. This permission allows you to activate the Save button in the "Create Assessment" popup. It can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission.
  • To see the Workflow Editor tile on the Dashboard and to edit or create schemes that govern the workflow through an assessment, you need "WorkflowRole" permission. The "WorkflowRole" permission is global only.

These permissions are granted by an ESMS administrator at the Security tile, Users tab, Global Permissions and/or User Workspace section.

Create Assessment

To add a new assessment, activate the Create Assessment button above the Assessment table.

Next you will enter data about the assessment. If applicable, this will include assigning individuals to permissions roles specified in the scheme.

Enter Assessment Data - All Assessments

All assessments need the following at the "Create Assessment" popup:

  1. Site. To initiate selecting a site, first activate the link icon () at the left side of the field. A new, smaller popup will appear.

    Note: Only one site may be selected for an assessment.

  2. At the "Organization Selectors" popup, do one of the following:
    1. Select one or more organizations to narrow the search field for sites.
    2. Alternatively, mark the Select All checkbox.
  3. Activate Next. A list of sites, each with a checkbox next to it, will be constructed. Depending on the number and size of the selected organization(s), it may take some time for the list of sites to appear.
  4. Check one and only one box to select the site.
  5. Activate Close. The popup will close and you will be back at "Create Assessment."
  6. Domain. Select a domain from the dropdown list.
  7. Budget. Enter the budget amount for the assessment here.
  8. Name. Enter a name for the assessment.
  9. Charter. Some text for the assessment's charter must be entered here. You can later edit or expand on it at the Details tab in the Charter stage.
  10. Planned Assessment Date. Enter or select the assessment date here.

    Tips: To select, navigate into the field box and activate the larger down arrow to bring up a calendar tool. To clear the date, navigate into the field box and activate the "x" (the "x" will appear only if a date has been entered or selected).

  11. Do one of the following based on whether the Workflow tool will be used for the assessment:
    1. If Workflow will not be used, skip to Save and Create in this topic.
    2. If Workflow will be used for the Assessment, continue at Enter Workflow Data.

Enter Workflow Data

Note: If Workflow will not be used for the assessment, skip to Save and Create after completing the steps above in Enter Assessment Data - All Assessments.

  1. To continue, you must first mark the Enabled checkbox to activate the remaining Workflow fields. (Shaded fields will become white.)

  2. Scheme. Select a scheme from the dropdown.

    3. IMPORTANT: If you need to edit a scheme in order for it to appropriately guide the workflow of a given assessment, make sure to do all edits necessary BEFORE assigning the scheme to the assessment at the "Create Assessment" popup. The assessment will function from a snapshot of the scheme taken at the time of creating the assessment, and will not reflect any changes made later to the scheme.

  3. Pre-Assessment Template. Select a pre-assessment template from the dropdown.

    Productivity Tip: For selecting this template and the next one, you can type the first part of the template name in the blank field, to bring up a match.

  4. Assessment Template. Select a assessment template from the dropdown.
  5. Permissions. Do one of the following based on whether the Permissions button is grayed out or active:
    1. If the Permissions button is grayed out, no permission roles need to be assigned. Skip to Save and Create.
    2. If the Permissions button is active (not grayed out), this means that the scheme you selected requires one or more people to be designated for roles that approve transitions from one stage to the next in the workflow. See the following list of instructions for how to designate permissions roles.

Permissions

Note: An assessment that does not use Workflow (and thus has no scheme) will not need Permissions.

Depending on the scheme selected, role permissions may need to be set for the new assessment. If the Permissions button near the bottom of the "Create Assessment" popup is active (not grayed out), this means that the scheme you selected requires one or more people to be designated for roles that approve transitions from one stage to the next in the workflow.

If you need to know more about Workflow permissions or how to manage them, see the topic Manage Workflow Permissions in the documentation about the Workflow editor.

To assign users to permissions roles,

  1. Activate the Permissions button at the bottom of the "Create Assessment" popup. The "Set Permissions" popup should appear.

    Note: Before you can set permissions, all the previous fields in the "Create Assessment" popup must be filled in.

  2. In the "Set Permissions" popup, activate the Add More button in a permissions category you want to add one or more users to. A popup will appear with a list of users to select from.
  3. If the user list spans multiple pages, you may need to bring appropriate names into view for selection as follows:
    1. Use the Name search field and the Go button to narrow down the selection list.
    2. As needed, use the pagination display at the bottom of the popup to step through the list of users.
  4. Select one or more users directly from the list as displayed in the popup. Activate an unselected name to select it, and activate a selected name to deselect it.
  5. Activate the Select button. You will be returned to the "Set Permissions" popup. You may repeat the previous three steps and this one as needed to select more users into the category.
  6. As needed, move on to another category in the "Set Permissions" popup and repeat the process, until you are finished with selections.
  7. When permissions are set for all desired users, activate the close button in the "Set Permissions" popup. The "Set Permissions" popup will close, and your selections will be retained.

    IMPORTANT: All permissions must be set at this stage before saving.

  8. Next, be sure to save and create the assessment (or cancel, if desired), as described in Save and Create.

Save and Create

To save the assessment data and create the assessment, activate the Save button at the lower right of the "Create Assessment" popup.

Result: If all needed information has been entered, the assessment will appear on a row of the Assessments table.

Next Step

Next you can select the assessment in the table and you will be brought to the Details tab of the Charter Stage. To access other tabs and stages, use the Workflow navigation tool (see Work Plan Navigation for instructions).
