About Cost Books

Enterprise SMS has an embedded cost module, used in the critical areas of work planning and condition assessment, that assigns cost values to inventory. The data used by the cost module is contained in one or more cost books.

Default costs are provided within Work Actions, and are all that is required. Optionally, a customized inflation book can be used in place of the default costs. Multiple customized cost books can be associated with different Organizations. This topic explains how to take full advantage of Cost Books.

Enterprise SMS's embedded cost module contributes to multiple Enterprise SMS functions:

  • Inventory: Provides itemization costs for all levels of the Enterprise SMS hierarchy.
  • Condition index calculations: Condition costs are used as weighing factors in rolling up CIs in the inventory hierarchy.
  • Work planning: For work planning, the cost module provides initial programming cost estimates for proposed maintenance, repair, and rehabilitation projects.

The cost data used by the cost module is contained in one or more cost books. A given Organization can have many defined cost books. Multiple cost books can also be associated with different Organizations. A single book can be assigned to an Organization for rollups, while the remaining books can be used for the Work Analysis process.

Cost book data is determined by and based on inflation data compiled from previous years. Cost book data for future years can be created with the help of Inflation Books. Refer to the About Inflation Books topic for additional information.

If an Organization does not have a cost book assigned to it, the Organization's assigned cost book is used for rollups. If no book is found to be assigned to an Organization, the process will run the course of the organizational hierarchy until a parent Organization with an assigned book is found.

Cost Data

The cost data used by Enterprise SMS's cost module is contained in one or more cost books. Multiple cost books can be associated with different Organizations as desired (see Assign a Cost Book to an Organization).

Default Cost Book

The Default Cost Book is found within Work Actions, which establish default costs for any combination of specifications not overridden by a custom cost book.

Custom Cost Book(s)

You can manage cost data by using custom cost books. To create custom cost data, you can either:

  1. Edit an existing cost book, or
  2. Create one or more new cost books. You can use an existing cost book as the starting point for the new cost book.

By assigning a custom cost book to an Organization, you can control the scope of that cost book, applying custom values to either all or part of the cost book.

Features and Search Bar

The following items can be viewed and are available for editing on the Features and Search Bar:

  • Work Action: The process(es) being executed, and the cost book item listed under Specification.
  • Specification: The item to be edited within the cost book.
  • Activity: The Global Repair, Replace or Demolish action required for the specified item.
  • Object Type: Identifies the kind of item found within the cost book.
  • Domain: The domain applied to the cost book, such as Utility-Gas, Waterfront, or Facility-Buildings.
  • Description: Identifies the cost book item.
  • Level: The Building or Component-Section required for the specified cost book item.
  • Parent: The parent Organization for the specified cost book item.
  • Current Cost: The most recently calculated value of the specified cost book item.
  • Original Cost: The earliest known calculated value of the specified cost book item.
  • Cost Year: The year used to calculate an item's inflation data.
  • UoM: The Unit of Measurement for the selected item(s).
  • Minimum Cost: The base-level value of the cost book item.
  • Materials: The value of materials used to complete the cost book item.
  • Labor: The value of the work used to execute the cost book item.
  • Equipment: The value of equipment used for the cost book item.
  • Bare Total: The estimated total value of the cost book.
  • Source: The foundation of the cost book.
  • Frequency: Describes how often the cost book data is updated.
  • Crew: The number of people assigned to the cost book.
  • Daily Output: The cost book's output value per day.
  • Total Output: The cost book's overall value for the fiscal year.
  • Remove: Select this option to remove the cost book.
