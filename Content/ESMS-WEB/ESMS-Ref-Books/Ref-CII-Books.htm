About CII Books

The Component Importance Index, or CII, is used to rank the relative importance of components against one another. One example is its use in Work Configuration to hold different Component-Sections to different thresholds, or to prioritize them differently based on relative importance.

Permissions Note: Depending on the operation (such as adding vs. deleting a book), configuring reference data requires Work Planner or Master Planner permission for the Organization or Site in conjunction with the reference book. The capacity to assign a custom reference book to an Organization or Site depends on your permissions for that Organization or Site.

CII Data

The Component Importance Index data used by the CII modules is contained in one or more CII books. Multiple custom CII books can be associated with different Organizations or Sites.

Default CII Book

The default CII book within Enterprise SMS is named "Reference," and it sets all items to a CII of 0.50. It cannot be edited.

You can choose to depend exclusively on the data in "Reference," or you can create one or many custom CII books.

Custom CII Book(s)

To create custom CII data, you can:

  1. Edit an existing CII book (except for Reference), or
  2. Create one or more new CII books. You can use Reference, or another existing CII book, as the starting point for the new CII book.

By assigning a custom CII book to an Organization or Site, you can control the scope of that CII book, applying custom values to all or part of the Organization or Site.

Assign a CII Book to an Organization or Site

You can create multiple CII books, and assign a separate book to any Organization or Site within your permissions scope, as desired. This is done as follows:

  1. In Inventory mode, navigate to and select the Organization or Site to assign the CII book to.

CII Book Tasks

Tasks you can perform are:

  • Add a Custom CII Book
  • Edit CII Books
  • Delete a CII Book
  • Assign a CII Book to an Organization or Site
