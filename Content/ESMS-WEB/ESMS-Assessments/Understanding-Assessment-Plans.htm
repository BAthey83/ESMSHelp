Understanding Assessment Plans

This topic introduces tasks you can initiate at the assessment plans screen and provides links to greater detail. To reach the assessment plans screen, activate the Assessment Plans tile on the ESMS Dashboard.

What Can I Do Here?

At the Assessment Plans screen, you can open an existing assessment plan listed in the table (see View Assessment Plans or Edit Assessment Plan), or (with appropriate permissions) create a new assessment plan (see Add Assessment Plans)

Note: Only assessment plans you have permission to view and/or work with will appear in the table.

In ESMS, you can create an assessment plan either with or without using the ESMS Workflow Engine tool ("Workflow"). Workflow is a tool that applies a custom-designed scheme of specific steps and approvals to your assessment plan process. Each type of assessment plan is described below, at About Creating An Assessment Plan Without Workflow and at About Creating An Assessment Plan Using Workflow.

Permissions Needed for Assessment Plans/Workflow

  • To edit or view information in stages of an assessment plan after it has been created, you need the ESMS permission named "Assessment Plan Editor." This can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission, to edit or view any assessment plan.
  • To create an assessment plan, one either with or without workflow, you need "WorkplanRole" permission. This permission allows you to activate the Save button in the "Create Plan" popup. It can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission.
  • To see the Workflow Editor tile on the Dashboard and to edit or create schemes that govern the workflow through an assessment plan, you need "WorkflowRole" permission. The "WorkflowRole" permission is global only.

These permissions are granted by an ESMS administrator at the Security tile, Users tab, Global Permissions and/or User Workspace section.

About Creating An Assessment Plan Without Workflow

If you create a new assessment plan without enabling Workflow, the plan will have two stages: Charter and Assessment.

Assessment Plan Setup (No Workflow)

Before proceeding to the stages of the assessment plan, there is information to be defined when you create the assessment:

  • Name the assessment plan
  • Specify the site the assessment plan will be associated with
  • Select a domain
  • Set a budget (this can be changed at the Charter stage)
  • Draw up a charter for the assessment entry (this can be edited at the Charter stage)
  • Set a planned assessment date (this can be changed at the Charter stage)
  • Assign users to PM, engineering, and/or reviewer roles

Charter Stage (No Workflow)

When Workflow has not been enabled at the "Create Plan" popup, the Charter stage screen will display just one tab: Details. Here you can view and (sometimes) change details that were specified during Assessment Plan Setup (Using Workflow). Site and domain can not be changed.

Assessment Plan Stage (No Workflow)

When you navigate to the Assessment Plan stage using the Draft DSR button in the Workflow navigation,

you will be brought to a Work Items page where you can manually add, edit, and delete work items.

About Creating An Assessment Plan Using Workflow

This section, plus Description of the Workflow Process, describes creating an assessment using the Workflow feature provided by the ESMS Workflow Engine tool.

If Workflow is enabled during assessment plan creation (this is done by marking the Enabled checkbox at the Workflow entry), then a work scheme must also be selected during assessment plan creation. The scheme will guide progress through the assessment plan, from resource planning to importing work items created in the field to including those work items into work packages.

If an appropriate scheme is not available for selection, there are two alternatives:

  1. Edit an existing scheme. If an existing scheme can be tweaked to suit, this can be done at the Workflow Editor tile.

    2. IMPORTANT: If you need to alter an existing scheme, make sure to do so before selecting the scheme for your assessment plan. When a scheme is selected for an assessment plan, that assessment plan operates from a snapshot made of the scheme at that point in time. This protects users from surprise changes to a scheme they are using.

  2. Add a new scheme. Alternatively, a brand new scheme can be added at the Workflow Editor tile. Using the Workflow Editor, you can design the sequential stages of creating an assessment plan using selected parameters.

Begin at the topic Understanding the Workflow Editor for information about editing or creating a scheme.

Description of the Workflow Process

A workflow process is defined by (1) activities and (2) transitions that link the activities together. The activities for creating an assessment plan are grouped into six stages, described in Stages. The flow is governed by a scheme.

Stages are defined at the Assessment Plans tile on the ESMS Dashboard, and are listed farther below in this topic. The work flow scheme is defined at the Workflow Editor tile on the ESMS Dashboard.

Assessment Plan Setup (Using Workflow)

Before designing the stages of the assessment plan, there is information to be defined when you create the assessment plan:

  • Name the assessment plan
  • Specify the site the assessment plan will be associated with
  • Select a domain
  • Set a budget (this can be changed at the Charter stage)
  • Draw up a charter for the assessment plan entry (this can be edited at the Charter stage)
  • Set a planned assessment date (this can be changed at the Charter stage)
  • Select a workflow scheme
  • Select a template for the pre-assessment plan questionnaire
  • Select a template for the field assessment plan questionnaire
  • Assign users to PM, engineering, and/or reviewer roles

For more detail, see the topic Add Assessment Plans.

Stages

The following six stages can be used to design the workflow for an assessment plan:

1. Charter. At the Charter Stage:

  1. A planned assessment date is or will have been set.
  2. A budget is or will have been set.
  3. A charter is or will have been drawn up.
  4. The template to be used at the Pre-Assessment Plan stage and the template to be used for assessment plans will have been selected.
  5. At the Assets tab, applicable inventory assets are selected.
  6. At the Groups tab, any applicable groups are selected.
  7. The Pre-Assessment Team is assigned at the Pre-Assessment Team tab. Each assessment plan team member will have at least one role or scope such as civil, electrical, or mechanical.

2. Pre-Assessment Plan. At the Pre-Assessment Plan Stage:

  1. If there are documents concerning the assessment plan, or documents to be associated with assets selected for assessment, they can be uploaded at the Documents tab at this stage.
  2. Questions on the pre-assessment plan questionnaire are answered at the Questionnaire tab and can be edited at either the Questionnaire or Questionnaire Summary tab.
  3. The Field Assessment Plan Team is assigned at the Field Assessment Plan Team tab. Each assessment team member will have at least one role or scope such as civil, electrical, or mechanical.

3. Review. At the Review Stage, the following can be reviewed but not altered:

  1. The charter.
  2. Assets associated with the assessment plan.
  3. Groups associated with the assessment plan.
  4. The list of any associated documents.
  5. Pre-assessment plan and field assessment plan team memberships.

4. Desktop Sync / Assessment. When the review is complete and any needed changes to the assessment plan have been made, you will:

  1. Synchronize to ESMS Desktop/Tablet.
  2. Exit to ESMS Desktop or Tablet to perform the assessment plan and generate work items.
  3. Re-synchronize after assessment, to bring assessment plan results and work items into ESMS Web.

5. Post-Assessment. At the Post-Assessment Plan Stage:

  1. The opportunity is provided here for people to review the assessment results, if the workflow design includes such a review.
  2. Work item details can be edited.
  3. Questionnaire answers can be edited.
  4. The DSR (work items) is approved or rejected. If approved, the assessment proceeds to the Packages stage.

6. Packages. At the Packages Stage

  1. Work packages can be created.
  2. Work items are assigned to work packages.
  3. Once the set of work packages has been approved, the work package stage is complete.

Transitions

Transitions between the stages are governed by the user-designed scheme. Any actions necessary for transition to the next stage are shown in the workflow tracking area at the bottom of the screen at each of the stages.

Scheme

A scheme is a definition of permissions and actions needed to transition from one stage to the next. The scheme customizes the workflow, and more than one scheme may be available to users to select from.

In everyday use of Workflow a scheme will be selected, not constructed. If you need to construct a scheme, start with the topic Understanding the Workflow Editor .

About Using the Workflow Process

Once the workflow process and the stages have been established, the information at each stage's tabs can guide team members in their roles and responsibilities. For example, the Pre-Assessment Plan Team tab at the Charter stage shows who is responsible for filling out the pre-assessment plan questionnaire in the Pre-Assessment Plan stage.

At each stage of the workflow process, the Workflow section at the bottom of the screen will vary in appearance depending on the scheme and other factors. If there are actions to be taken to proceed (or revert) to another stage in the workflow, this will be indicated by one or more buttons in the Workflow section. For example:

When a user with appropriate permissions approves the transition and confirms it in the popup that appears when the Approve to Pre-Assessment Plan button is activated, then the workflow progresses to that stage.

About Tracking Workflow Status

In an Assessment Plan stage, the name of the current state is displayed near the top of the page (for example "Project Planning" in the screen shown at About Using the Workflow Process). State is a term used in the Workflow editor to refer to an element in the Scheme.

You can activate either the state name, or the clock icon next to it, to see the "Transition History" popup listing all actions that have been taken to move the workflow along, such as approvals, comments, or transitions to another state/stage. You might also receive alerts of these actions on the main menu bar (the bell icon will have a red circle on it showing the number of alerts), if the Notifications Enabled checkbox was marked in the Required Permission section of the "Transition" popup when the scheme transition was designed or edited (see Understanding the Workflow Editor .

Assessment Level Task

The following assessment plan tasks can be initiated at the Assessment Plans screen:

  • View Assessment Plans
  • Add Assessment Plans
  • Edit Assessment Plan
  • Delete Assessment Plan

Instructions for navigating to and in an assessment plan are in the topic Assessment Plan Navigation.

Stage Level Tasks

Other changes to an assessment plan can be made at the appropriate stage within the assessment plan. The stages of a workflow-enabled assessment plan are determined by the scheme being used. Currently, the workflow provided with ESMS consists of the following stages, which are described at Workflow Stages:

  • Charter Stage
  • Pre-Assessment Plan Stage
  • Review Stage
  • Desktop Sync / Assessment Stage
  • Post-Assessment Plan Stage
  • Packages Stage
