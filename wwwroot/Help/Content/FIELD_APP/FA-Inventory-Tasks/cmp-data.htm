Component Data

Data Required when Adding a Component

When you select Add Component in the ribbon at the Assets tab, a popup will appear where you need to enter the following items of required data:

  • Domain - select a domain from the dropdown list

  • Component Type - select a component type from the dropdown
  • Name - enter a name for the component
  • Quantity - enter the quantity/length
  • Units - enter a unit of measure from the dropdown

  • Groups - this is not required if you don't wish to associate the component with any groups.

 

 

Additional data is described below under Full Component Data.

Full Component Data

Component data is divided into these categories:

  • Basic properties
  • Classification properties
  • System properties
  • User defined properties
  • Groups

In Add Component, only Groups are shown in a separate fieldset. However, in Edit Component, all of these categories will be shown in the content area.

Some of the categories may be minimized on initial view. To expand, click the downward-facing caret to the left of the fieldset title. If the fieldset contains no data, the only change will be the caret change to .

When adding a component, you can assign the following fields:

  • Domain. Select from the dropdown.
  • Component Type. Select the from the dropdown.
  • Name. Provide a name for the component. If you also use a component number (that is not the UNIFORMAT code for the component), enter that in this field as well.
  • Site. This is automatically assigned for you.
  • System/Parent Component. This is automatically assigned for you. The "System" field will appear when you create a component directly under the system; the "Parent Component" field will appear when you create a component under another one (only one sublevel is allowed).

Additional basic information is available when you are editing the component:

  • Real Property (if shown). This is the Real Property Unique Identifier RPUID). Click on the magnifying glass icon or anywhere in the field to bring up a list of Real Property selections.
  • Date Installed. By default, this is the date the component was entered into Enterprise SMS; you can change it when you edit the component.
  • Operational Status: "In Service" or "Out of Service". By default, Enterprise SMS assigns "In Service" when the component is entered into Enterprise SMS; you can change it when you edit the component.

You can also edit the Name field.

When editing a component, you can edit the following classification properties if they are shown:

  • UNIFORMAT. A UNIFORMAT code may have been assigned at creation of the component.

To change this code,

  1. Click the magnifying glass icon or anywhere in the field to show a tree of UNIFORMAT options.
  2. Select the desired code. The code will be changed and the tree display will close.

To close the tree display without making a change, do one of the following:

  1. Click the magnifying glass icon again.
  2. Click the "x" at the upper right of the popup containing the UNIFORMAT tree.

Expanding User Defined Properties will display any user defined properties that have been established for the component. To expand, click the downward-facing caret . If the User Defined Properties fieldset contains no data, the only change will be the caret change to .

 

Groups (Optional)

Expanding Groups will display any available groups that the component can be associated with. To expand, click the downward-facing caret . If groups have not been established, the only change will be the caret change to .

Using the dropdown lists provided for each group area, select every group that the component is associated with.
