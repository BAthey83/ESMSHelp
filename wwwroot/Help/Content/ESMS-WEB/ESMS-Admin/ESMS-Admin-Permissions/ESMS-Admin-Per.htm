Permissions Scopes

Each user is assigned to one or more Roles. Roles are allowed varying degrees of Permission Scopes within Enterprise SMS.

Permission Scopes include:

  • Global: Grants access to all data from Organizations, Users, and Catalogs.
  • Organization: Grants access to all Organization level data, including all Sites and Groups within that Organization.
  • Site: Grants access to Site level data, including all Groups within the Site.
  • Group: Grants access to data related to all of the Facilities within a specified Group.

Creating User Accounts

Initiating the Set-Up Process

Potential Enterprise SMS users can initiate the account set-up process by completing the proper form(s) and receiving the necessary approvals. For more information on the Account Creation process, refer to the Introduction to User Permissions.

User Administration

Users who have Administrator permissions can create Permissions and assign them to other users. Enterprise SMS has three types of Administrator permissions: Security Administrator, Organization Administrator, and Site Administrator. 

Administrator permissions should only be given to users who need to manage other users. 
			There are additional forms/approvals required to obtain Administrator permissions.

How to Assign Security Administrator Permissions to a User

A Security Administrator has global access to everything in Enterprise SMS. Only Security Administrators can assign another user to be a Security Administrator.

  1. Log in to Enterprise SMS and navigate to Administration > Users.
  2. Type the name of the user who will be granted Administrator permissions into the search bar.

    3. Note: The user must complete a user account creation before performing the following steps.

  3. Click on the user's name.
  4. Switch the toggle next to "Security Administrator" to the "On" position.
  5. Click "Save."

How to Assign Organization and Site Administrator Permissions to a User

Organization and Site Administrator permissions are identical to that of a Security Administrator. However, they are specific to an Organization or Site assigned to the permission.

ESMS uses "inherited" permissions. As a result, a user's permissions assigned at the Organization level will also be applied to all Sites under the same Organization. For example, if a user has Administrative permissions at the Organization level, then the user will also have Administrative permissions for all Sites under the Organization.

Note: A user must be a Security Administrator to be able to assign a user to be an Organization or Site Administrator.

  1. Log in to ESMS and navigate to Administration > Users.
  2. Type the name of the user who will be granted Administrator permissions into the search bar.

    3. Note: The user must complete a user account creation before performing the following steps.

  3. Click on the user's name.
  4. Select the "+" button next to User Site Administration or User Organization Administration.
  5. Type the name of the Site or Organization.
  6. Click the desired Site or Organization.
  7. Select "Save."

How to Lock/Unlock a User's Account

Note: A user must have Security Administrator permissions to lock/unlock a user's account.

  1. Log in to ESMS and navigate to Administration > Users.
  2. Type the name of the user who will be granted Administrator permissions into the search bar.

    3. Note: The user must first complete a user account creation.

  3. Click on the user's name.
  4. Switch the toggle next to "Locked" to lock/unlock a user's account.
  5. Click "Save."

How to Update a User's Name and/or Email

Note: A user must have Security Administrator permissions to update the user's name or email address.

  1. Log in to ESMS and navigate to Administration > Users.
  2. Type the name of the user who will be granted Administrator permissions into the search bar.

    3. Note: The user must complete a user account creation before performing the following steps.

  3. Click on the user's name.
  4. Update the user's email address and/or first/last names in the text input fields.
  5. Click "Save."

Creating Team Permissions

Groups of users can have permissions created and assigned to them in Enterprise SMS with Teams. Teams can be created at the Organization, Site, and Group levels.

Permissions assigned at a higher level will also inherit the lower level permissions. For example, if a user is assigned Inventory Delete permissions, the user can also Create, Update, and Read. Ranking from high to low, the permission levels are as follows: Delete, Update, Create, and Read.

Note: Teams and Roles are identical. These terms may be used and referred to interchangeably throughout the topic.

Creating a Team at the Organization Level

Note: A user must have Security or Organization Administrator permissions to the Organization for which the team is being created.

  1. Log in to ESMS and navigate to Inventory > Organizations.
  2. Click on the desired Organization from the list.
  3. Click the Permissions tab.
  4. Click the "+" button.
  5. Type in the Team's name in the "Name" field.
  6. Set the permission(s) for Asset and Inventory using the drop-down menus.
  7. Use the toggle switches to turn the other permissions on/off.
  8. Click "Save."

Creating a Team at the Site Level

Note: A user must have Security or Organization Administrator permissions to the Organizationor Site, or Site Administrator permissions for which the team is being created.

  1. Log in to ESMS and navigate to Inventory > Sites.
  2. Click on the desired Site from the list.
  3. Click the Permissions tab.
  4. Click the "+" button.
  5. Type in the Team's name in the "Name" field.
  6. Set the permission(s) for Asset and Inventory using the drop-down menus.
  7. Use the toggle switches to turn the other permissions on/off.
  8. Click "Save."

Creating a Team at the Group Level

Note: A user must have Security Administrator permissions, Organization Administrator permissions for the Organization or Group, or Site Administrator permissions to the Site of the Group for which the team is being created.

  1. Log in to ESMS and navigate to Inventory > Groups.
  2. Click on the desired Group from the list.
  3. Click the Permissions tab.
  4. Click the "+" button.
  5. Type in the Team's name in the "Name" field.
  6. Set the permission(s) for Asset and Inventory using the drop-down menus.
  7. Use the toggle switches to turn the other permissions on/off.
  8. Click "Save."

Adding Users to a Team

Note: A user must have the appropriate Administrator level to add a User to a team as stated previously. This section is applicable to Organization, Site, and Group level teams.

  1. From the Permissions tab, click on the "˄" button next to desired team to expand the user's list. (It will become a "˅" button once activated).
  2. Click the "+" button next to "User Name."
  3. Type the user's name into the search bar.
  4. Click the user's name to be added to the team.
  5. Click "Save."

Permission FAQs

  • Q: Is there a way to limit what domain(s) a user can access? Is this handled outside of the Roles/Permissions page?
  • A: Not at the present time.
  • Q: What happens if a user is assigned conflicting permissions at the Organization and Site levels? (For example, the user is on a team with Delete Inventory permissions at the Organization level, but only Read Inventory permissions at the Site level).
  • A: The highest permission level overrules the others. In the above example, the user will retain Delete Inventory permissions.
