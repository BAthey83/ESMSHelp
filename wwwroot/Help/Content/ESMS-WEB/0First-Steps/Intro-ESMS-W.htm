What Can ESMS Web Do?

Using the Web version of Enterprise SMS, you can

  • import real property data
  • explore the data and search for specific data items
  • store and retrieve documents
  • generate work items based on criteria
  • run reports
  • administer permissions for individuals and teams (requires Team Leader or ESMS administrator permission level)

ESMS administrators can also

  • create component types and assign user-defined fields for the new types
  • create group types
  • create templates for customized assessments
  • create new user accounts

ESMS Web is not intended for

  • real property inventory entry
  • performing assessments

Do I Need ESMS Web, the ESMS Field Application, or Both?

Note: The ESMS field application offers two modes, Desktop and Tablet. Tablet functionality is restricted to performing assessments and navigating asset information.

Web/Field Coordination

The typical work flow interaction between ESMS Web and the ESMS field application typically follows these steps:

  1. Inventory and assessments are entered into ESMS Desktop. Assessments may alternatively be entered in the field using ESMS Tablet.
  2. Synchronization is performed between ESMS Desktop and ESMS Web.
  3. Work items are generated in ESMS Web, in the Work task area, by specifying threshold levels for condition index and remaining service life.

Topical Overview of Web and Desktop Capabilities

  • Real Property Inventory
    1. For searching and viewing real property inventory, ESMS Web will suffice if all the inventory you need is kept up-to-date on your Web instance by other parties.
    2. However, in order to add or change inventory entries, you definitely need to use ESMS Desktop in addition to ESMS Web.
  • Associating Documents with Components
    1. Documents such as images can be imported into ESMS and "attached" to the relevant components in inventory only in ESMS Web.
    2. ESMS Desktop also has functionality for storing documents in association with components. These are synchronized to and from ESMS Web via the server (see Integration Between ESMS Web, Desktop, and Tablet).
  • Associating Components with Groups
    1. ESMS Web has no tools for associating components with groups. This is done in ESMS Desktop.
    2. In ESMS Desktop, a component can be associated with one or more groups in the "Add Component" popup when creating a new component.
  • Assessments
    1. Assessments can be viewed in ESMS Web.
    2. To create assessments, you need to use the ESMS field application. Assessments can be entered into ESMS Desktop, or performed in the field using ESMS Tablet.
  • Work Items
    1. Work items can be generated in ESMS Web, in the Work task area, by specifying threshold levels for condition index and remaining service life.
    2. To enter a work item manually, you need to use ESMS Desktop.

Integration Between ESMS Web, Desktop, and Tablet

As shown in the diagram below, ESMS Web, ESMS Desktop, and (if applicable) ESMS Tablet are each connected to a central server. In the case of ESMS Web, synchronization of data to and from the server occurs automatically. With ESMS Desktop and ESMS Tablet, however, you must initiate a synchronization to (a) upload information (such as inventory that has been entered, or data on assessments that have been performed), or to (b) download Workflow specifications or the latest information from ESMS Web.
