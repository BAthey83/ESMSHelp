Understanding Assessments

This topic introduces tasks you can initiate at the Assessments screen and provides links to greater detail. To reach the Assessments screen, activate the  Assessments tile on the ESMS Dashboard.

What Can I Do Here?

At the Assessments screen, you can open an existing assessment listed in the table (see View Assessment or Edit Assessment), or (with appropriate permissions) create a new assessment (see Add Assessment).

Note: Only assessments you have permission to view and/or work with will appear in the table.

In ESMS, you can create an assessment either with or without using the ESMS Workflow Engine tool ("Workflow"). Workflow is a tool that applies a custom-designed scheme of specific steps and approvals to your assessment process. Each type of assessment is described below, at About Creating an Assessment Without Workflow and at About Creating an Assessment Using Workflow.

Permissions Needed for Assessments/Workflow

  • To edit or view information in stages of an assessment after it has been created, you need the ESMS permission named "Assessment Editor.". This can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission, to edit or view any assessment.
  • To create an assessment, one either with or without workflow, you need "AssessmentRole" permission. This permission allows you to activate the Save button in the "Create Assessment" popup. It can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission.
  • To see the Workflow Editor tile on the Dashboard and to edit or create schemes that govern the workflow through an assessment, you need "WorkflowRole" permission. The "WorkflowRole" permission is global only.

These permissions are granted by an ESMS administrator at the Security tile, Users tab, Global Permissions and/or User Workspace section.

About Creating a Assessment Without Workflow

If you create a new assessment without enabling Workflow, the assessment will have two stages: Charter and Assessment.

Assessment Setup (No Workflow)

Before proceeding to the stages of the assessment, there is information to be defined when you create the assessment:

  • Name the sssessment
  • Specify the site the assessment will be associated with
  • Select a domain
  • Set a budget (this can be changed at the Charter stage)
  • Draw up a charter for the Assessment entry (this can be edited at the Charter stage)
  • Set a planned assessment date (this can be changed at the Charter stage)
  • Assign users to PM, engineering, and/or reviewer roles

Charter Stage (No Workflow)

When Workflow has not been enabled at the "Create Assessment" popup, the Charter stage screen will display just one tab: Details. Here you can view and (sometimes) change details that were specified during assessment setup. Site and domain can not be changed.

Assessment Stage (No Workflow)

When you navigate to the Assessment stage using the Draft DSR button in the Workflow navigation,

you will be brought to a Work Items page where you can manually add, edit, and delete work items.

About Creating a Assessment Using Workflow

This section, plus Description of the Workflow Process, describe creating an assessment using the Workflow feature provided by the ESMS Workflow Engine tool.

If Workflow is enabled during assessment creation (this is done by marking the Enabled checkbox at the Workflow entry), then a work scheme must also be selected during assessment creation. The scheme will guide progress through the assessment, from resource planning to importing work items created in the field to including those work items into work packages.

If an appropriate scheme is not available for selection, there are two alternatives:

  1. Edit an existing scheme. If an existing scheme can be tweaked to suit, this can be done at the Workflow Editor tile.

    IMPORTANT: If you need to alter an existing scheme, make sure to do so before selecting the scheme for your ssessment. When a scheme is selected for a assessment, that assessment operates from a snapshot made of the scheme at that point in time. This protects users from surprise changes to a scheme they are using.

  2. Add a new scheme. Alternatively, a brand new scheme can be added at the Workflow Editor tile. Using the Workflow editor, you can design the sequential stages of creating an assessment using selected parameters.

Begin at the topic Understanding the Workflow Editor for information about editing or creating a scheme.

Description of the Workflow Process

A workflow process is defined by (1) activities and (2) transitions that link the activities together. The activities for creating an assessment are grouped into six stages, described in Stages. The flow is governed by a scheme.

Stages are defined at the Assessments tile on the ESMS Dashboard, and are listed farther below in this topic. The work flow scheme is defined at the Workflow Editor tile on the ESMS Dashboard.

Assessment Setup (Using Workflow)

Before designing the stages of the assessment, there is information to be defined when you create the assessment:

  • Name the assessment
  • Specify the site the assessment will be associated with
  • Select a domain
  • Set a budget (this can be changed at the Charter stage)
  • Draw up a charter for the assessment entry (this can be edited at the Charter stage)
  • Set a planned assessment date (this can be changed at the Charter stage)
  • Select a workflow scheme
  • Select a template for the pre-assessment questionnaire
  • Select a template for the assessment questionnaire
  • Assign users to PM, engineering, and/or reviewer roles

For more detail, see the topic Add Assessment.

Stages

The following six stages can be used to design the workflow for an assessment:

  1. Charter. At the Charter stage:
    1. A planned assessment date is or will have been set.
    2. A budget is or will have been set.
    3. A charter is or will have been drawn up.
    4. The template to be used at the Pre-Assessment stage and the template to be used for assessment will have been selected.
    5. At the Assets tab, applicable inventory assets are selected.
    6. At the Groups tab, any applicable groups are selected.
    7. The pre-assessment team is assigned at the Pre-Assessment Team tab. Each assessment team member will have at least one role or scope such as civil, electrical, or mechanical.
  2. Pre-Assessment. At the Pre-Assessment stage:
    1. If there are documents concerning the assessment, or documents to be associated with assets selected for assessment, they can be uploaded at the Documents tab at this stage.
    2. Questions on the pre-assessment questionnaire are answered at the Questionnaire tab and can be edited at either the Questionnaire or Questionnaire Summary tab.
    3. The Assessment team is assigned at the Assessment Team tab. Each assessment team member will have at least one role or scope such as civil, electrical, or mechanical.
  3. Review. At the Review stage the following can be reviewed but not altered:
    1. The charter.
    2. Assets associated with the assessment.
    3. Groups associated with the assessment.
    4. The list of any associated documents.
    5. Pre-assessment and assessment team memberships.

    You must back up to a previous stage to make any changes desired.

  4. Desktop Sync / Assessment. When the review is complete and any needed changes to the assessment have been made, you will:
    1. Synchronize to ESMS Desktop/Tablet.
    2. Exit to ESMS Desktop or Tablet to perform the assessment and generate work items.
    3. Re-synchronize after assessment, to bring assessment results and work items into ESMS Web.
  5. Post-Assessment. At the Post-Assessment stage:
    1. The opportunity is provided here for people to review the assessment results, if the workflow design includes such a review.
    2. Work item details can be edited.
    3. Questionnaire answers can be edited.
    4. The DSR (work items) is approved or rejected. If approved, the assessment proceeds to the Packages stage.
  6. Packages. At the Packages stage:
    1. Work packages can be created.
    2. Work items are assigned to work packages.
    3. Once the set of work packages has been approved, the work package stage is complete.

Transitions

Transitions between the stages are governed by the user-designed scheme. Any actions necessary for transition to the next stage are shown in the workflow tracking area at the bottom of the screen at each of the stages.

Scheme

A scheme is a definition of permissions and actions needed to transition from one stage to the next. The scheme customizes the workflow, and more than one scheme may be available to users to select from.

In everyday use of Workflow a scheme will be selected, not constructed. If you need to construct a scheme, start with the topic Understanding the Workflow Editor .

About Using the Workflow Process

Once the workflow process and the stages have been established, the information at each stage's tabs can guide team members in their roles and responsibilities. For example, the Pre-Assessment Team tab at the Charter stage shows who is responsible for filling out the pre-assessment questionnaire in the Pre-Assessment stage.

At each stage of the workflow process, the Workflow section at the bottom of the screen will vary in appearance depending on the scheme and other factors. About Tracking Workflow Status

In an assessment stage, the name of the current state is displayed near the top of the page (for example "Project Planning" in the screen shown at About Using the Workflow Process ). State is a term used in the Workflow editor to refer to an element in the scheme.

You can activate either the state name, or the clock icon next to it, to see the "Transition History" popup listing all actions that have been taken to move the workflow along, such as approvals, comments, or transitions to another state/stage (see sample below in the screenshot). You might also receive alerts of these actions on the main menu bar (the bell icon will have a red circle on it showing the number of alerts), if the Notifications enabled checkbox was marked in the Required Permission section of the "Transition" popup when the scheme transition was designed or edited in the Workflow Editor.

Assessment Level Tasks

The following assessment tasks can be initiated at the Assessments screen:

  • View Assessment
  • Add Assessment
  • Edit Assessment
  • Delete Assessment

Instructions for navigating to and in an assessment are in the topic Assessment Navigation.

Stage Level Tasks

Other changes to an assessment can be made at the appropriate stage within the assessment. The stages of a workflow-enabled assessment are determined by the scheme being used. Currently, the workflow provided with ESMS consists of the following stages, which are described at Stages:

  • Charter Stage
  • Pre-Assessment Stage
  • Review Stage
  • Desktop Sync / Assessment Stage
  • Post-Assessment Stage
  • Packages Stage
