What Is a Scheme?

A scheme is comprised of states and transitions:

  • A state represents the condition that the process is in at a specific time.
  • A transition defines how the process will move from one state to the next.

A scheme may also make use of parameters to store variables that are global to the scheme.

In the Workflow editor, a scheme's states and transitions are presented as a diagram. The states appear as boxes in the diagram. The transitions are represented by lines connecting the boxes. For clarity, each state should be named; transition names are optional.

Navigate to a Scheme

To view or edit a scheme's diagram and details, first enter the Workflow editor:

  1. Navigate to the ESMS Web Dashboard.
  2. Activate the Workflow Editor tile.

Instructions near the top of the screen say to "Please select a scheme"; this is done from the Workflow editor menu bar as follows:

  1. On the Workflow editor menu bar (which is located below the main menu bar), activate the Open Scheme icon, which looks like a partially open folder.
  2. In the "Open Scheme" popup, activate the dropdown arrow.
  3. Select the desired scheme from the list.

Note: the scheme will appear in the work area as soon as you select it in the dropdown list.

  1. If you wish, you can change your choice at this point by re-activating the down arrow and making a different selection.
  2. Activate the OK button to exit the popup.

State Details

To view or edit details about a state, activate the state box in the diagram and the corresponding "State" popup will appear with the details.

State details are fairly simple:

  • Name - Name of the state, which appears in the state box on the diagram. This can be changed.
  • Initial / Final - If the state is the first state in the work flow, the Initial checkbox should be marked; if the state is the last state in the work flow, the Final checkbox should be marked (this information is also indicated by colors in the diagram).
  • Code Action - The "Reset all approvals" code action allows an approval to be re-used for a subsequent transition without having to rename it. A "set X stage" code action creates a linking between the state and an assessment stage. Code actions are executed when the state is entered.

Transition Details

To see details about a transition, activate anywhere on the relevant connecting line in the diagram, and the corresponding "Transition" popup will appear with the transition's details. The "Transitions" popup is where the magic of the Workflow is designed in.

In the descriptions below, Required items are those required by the software. Your process may include items that are Optional to the software, but required for your workflow.

  • Name (Required) - This is the label that will show over the transition line in the diagram
  • From State (Required) - A transition connects two states; the "from" state is where the transition begins
  • To State (Required) - A transition connects two states; the "to" state is where the transition ends

Note: The same state may be entered at "From State" and "To State," in which case on the diagram you should see a small loop exiting and re-entering the relevant state

  • Type (Required) - The type will be one of the following:
    • Command - Transition happens when the user pushes a button
    • Auto - A rule decides when the transition happens
  • User Inputs (Optional) - Any approvals or comments needed or allowed from the user are listed here
  • Required Permissions (Optional for Command transition type; N/A for Auto transition type) - If the user to be executing a Command needs a particular permission type, this is listed here
  • Commands (Optional for Command transition type; N/A for Auto transition type) - For a Command type transition, these are things that must be done before the transition can occur. Further detail needed for commands are:
    • Command Name (Required) - A descriptive name such as "Service Control Approve or Reject" should be assigned to the command
    • Parameters - Example parameters for commands are "Approval: Approved" "Approval: Rejected" or "IsEnabled: true"
  • Conditions (Optional for Auto transition type; N/A for Command transition type) - Here is where conditions–if any–are listed that determine when an Auto type transition happens; if the transition type is Auto and no conditions are listed, the transition will happen automatically.
