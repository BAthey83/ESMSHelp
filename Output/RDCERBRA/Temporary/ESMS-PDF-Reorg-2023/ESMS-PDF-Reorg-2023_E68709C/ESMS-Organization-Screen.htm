This topic shows how to add a Site and describes the initial information to be entered. The topic Edit Inventory Site Dataon page 1 describes additional information that can be entered for the site.

Copy Factors from a Listed Location (Optional)

You are given the opportunity to copy and auto-apply Site location factors (Area Cost Factor, seismic zone, and HVAC zone) from an existing location. The Enterprise SMS database contains this data for most major cities in the United States, as well as for Department of Defense installations.

To copy an existing location's factors into your site, mark the Use factors from following location checkbox and select a location from the dropdown lists.

Copy factors from an existing location can also be done later by selecting Lookup Location from the Site toolbar.

Note: Until you set location factors either by copying from a location or manually editing them at the Site data page, they will all be set by default to 1.00.

Generate Report at Building Data Screen

Scope of Report

When you generate a report while you are at the Site level, the report will contain only data that applies to Buildings in the currently selected Site.

How to Access

To access a report from a Site's inventory data screen,

  1. Select a Site from the Site Search screen.
  2. Open the selected Site to observe the report data.
  3. Additionally, you can also apply one or more filters at the Site Search screen to obtain a Site's report data.

Generate Report from Main Menu

Scope of Report

By default, unless you have used filters to further restrict what is included in the report, the scope of the report will be the Organization, Site, Building, System, Component, or Component-Section where you are located within the Enterprise SMS framework.

How to Generate

To generate a report from the main menu,

  1. Select the Reports button from the main menu.
  2. Choose an Organization and Report from the Reports popup menu.
  3. Select the Sites tab.
  4. Click the Site dropdown menu to pick a site in order to access the desired report.
  5. Optionally, you can then select one or more filters at the Organization or Domain levels. For example, you can select the Fuels domain to only show Sites which host Liquid Fueling and Dispensing Facilities.
  6. After setting any desired filters, and selecting a Sort order, you can then access the Report data.
  7. From the More Options popup menu, you can choose to export the data as a CSV file or show it as a table. You can spotlight data, get insights, sort data in ascending or descending order, or sort by installation name, site name, facilities, or PRVPlant Replacement Value. This number will be the aggregate of the PRV of all inventory items contained within the current inventory item..

Delete a Site

This topic shows how to delete a Site, which also deletes all other records (Buildings, Systems, etc.) contained in the Site.

Permissions Note: This action requires Administrator permissions. If the inventory item is empty of any contents, then the deletion may also be performed by a Master Planner with permission for the relevant location in Inventory.

To delete a Site, navigate to it in the Site Search window and do the following:

  1. Select the Site you wish to delete in the Site Search window, and then click the Delete icon at the far right.
  2. Alternatively, you can select the Organizations button under the Inventory dropdown menu. Select your Organization, and then select the General tab, followed by the Sites menu. Here, you can select and delete the Site(s) as you wish.

WARNING: Deleting a Site deletes all inventory (Complexes, Buildings, Systems, Components, and Sections) in that Site. This is a significant step to take and should only be done when certain that you wish to clear the entire inventory of the selected Site.

Best Practice Recommendation: Making frequent backups of the inventory database will protect you from significant data losses if an unintended deletion is performed.

Working with Sites

A Site is the level in Enterprise SMS that ties your real property inventory to a physical location. This topic briefly introduces the features of a Site and how to use it in organizing your real property inventory. It also provides links to the operations that can be performed at the Site level.

Your Enterprise SMS inventory can contain Buildings or facilities at a number of different geographic locations, or Sites. If this is the case, multiple Sites will be needed, one for each geographic location. Alternatively, if all Buildings or Facilities to be managed are in the same geographic location, one Site will generally suffice.

The Importance of Geography to the Site Level

Knowing the geographic location of a Building is extremely important; therefore it is necessary to create a Site record for each geographic location before you can add Buildings to the database.

When creating a Site, keep in mind that the Site level is not simply a method for subdividing inventory. It is the level where geographic factors are incorporated into Enterprise SMS calculations, and therefore a Site needs to be confined to a relatively limited geographic area. A larger area can be covered by an Organization made up of multiple Sites.

Site-Level Properties

Several properties of Buildings are determined or affected by the Building's geographic location (Site), most critically the Area Cost FactorA multiplier that adjusts national average labor/material/equipment costs to local costs. For example, in a remote area with high shipping costs and a tight labor market, the Area Cost Factor might be 1.3. The Area Cost Factor is generally a number between 0.8 and 1.2 for continental U.S. locations, but it can be even greater than 2.0 in high-cost geographic locations..
