View Work Item

To view a work item,

  1. In the navigation pane, select the component that the work item is associated with.
  2. At the assessment plans tab, activate Work Item in the ribbon. The component's Work Item tab will appear in the content pane.

    CAUTION: The work item's entry fields will be active. If you only wish to view, refrain from typing.

The listing below explains the data fields in a work item.

  • Description - Brief (maximum of 50 characters), identifying description of the work to be performed. After you finish entering this description, it will also appear under "Work Items" near the top of the content pane.
  • Reference - This corresponds to "Work Request ID" in legacy SMS. This item of data is not used by Enterprise SMS in any analyses; it's a field that allows entry of whatever work request or other identifying number is used by your organization for the work item.
  • Cost Amount - The estimated cost to perform the work included in the work item. As you enter the cost amount it will also appear under "Work Items" near the top of the content pane.
  • Date - $$not a match to legacy builder
  • Current Situation - Description of the component's current situation or condition that merits the repair or replacement called for in the work order.
  • Impact - Description of the adverse impact resulting from the current situation.
  • Scope of Work - This is where more detail about the work to be done is entered. $$need definition, worded neutrally as a noun phrase.
  • Regulatory Requirement - $$need definition, worded neutrally as a noun phrase.
