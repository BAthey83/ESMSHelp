Edit Scheme States

To view or edit details about a state, activate the state box in the diagram and the corresponding "State" popup will appear with the details.

State details are fairly simple:

  • Name - Name of the state, which appears in the state box on the diagram. This can be changed.
  • Initial / Final - If the state is the first state in the work flow, the Initial checkbox should be marked; if the state is the last state in the work flow, the Final checkbox should be marked (this information is also indicated by colors in the diagram).
  • Code Action - The "Reset all approvals" code action allows an approval to be re-used for a subsequent transition without having to rename it. A "set X stage" code action creates a linking between the state and an assessment stage. Code actions are executed when the state is entered.

Delete a State from a Scheme

To delete a state or transition within a scheme, select the state or transition in the diagram and activate the Delete key on the keyboard. Make sure to save your change(s) by activating the Save icon button on the Workflow editor menu bar.

WARNING: Deleting a state will delete all transitions connected to it. If you want to preserve one or more of these transitions, then before you delete the state, change the beginning ("From") and end ("To") points of the transition(s) as needed so that they bypass the to-be-deleted state entirely.

Note: If you just added the state you want to delete, simply activating the Delete key is sufficient. However, this delete procedure goes back only one step, to the most recent state or transition you added. Elements placed in the diagram earlier need to first be selected before you activate Delete.
