Cost Books

Enterprise SMS has an embedded cost module, used in the critical areas of work planning and condition assessment, that assigns cost values to inventory. The data used by the cost module is contained in one or more cost books.

Default costs are provided within Work Actions, and are all that is required. Optionally, a customized cost book can be used to override the default costs. Multiple customized cost books can be defined for different Organizations. This topic explains how to take full advantage of cost books.

Introduction to Cost Books

Enterprise SMS's embedded cost module contributes to the following Enterprise SMS functions:

  • Inventory: Provides itemization costs for all levels of the Enterprise SMS hierarchy.
  • Work Planning: For work planning, the cost module provides initial programming cost estimates for proposed maintenance, repair, and rehabilitation projects.

The cost data used by the cost module is contained in one or more cost books. A given Organization can have many defined cost books. A single book can be assigned to an Organization for rollups, while the remaining books can be used for the Work Analysis process.

Cost book data is determined by and based on inflation data compiled from previous years. Cost book data for future years can be created with the help of Inflation Books. Refer to the Inflation Books topic for additional information.

If an Organization has a cost book assigned to it, the cost book will be used for rollups. If no book is found to be assigned to an Organization, the process will run the course of the organizational hierarchy until a parent Organization with an assigned book is found.

Access to Cost Books

To access the Cost Books screen,

  1. Open the Reference Books drop-down menu.
  2. Select the Cost button, as highlighted above.

Cost Data

The cost data used by Enterprise SMS's cost module is contained in one or more cost books. Multiple cost books can be defined for different Organizations as needed (see Assign a Book to an Organization).

Default Cost Book

The Default Cost Book is found within Work Actions, which establish default costs for any combination of specifications and work activities not overridden by a custom cost book.

Custom Cost Book(s)

You can manage cost data by using custom cost books. To create custom cost data, you can either:

  1. Edit an existing cost book, or
  2. Create one or more new cost books. You can use an existing cost book as the starting point for the new cost book.

By assigning a custom cost book to an Organization, you can control the scope of that cost book, applying custom values to either all or part of the cost book.

Features and Search Bar

The Features and Search Bar is where you can review and edit data for existing Cost Books.

  • Save Changes: Select the floppy disk icon on the left to retain your edits to an existing Cost Book.
  • Filter: Select the center icon to add a filter to your Cost Book search results.
  • Select Columns: Choose the triple-dash icon on the right to select which data columns to view in your Cost Book.

Fields on the Features and Search Bar can be customized as needed. To add or remove fields from the Features and Search Bar, click the Select Columns button, as indicated by the triple-dash icon highlighted above. The Select Columns to View popup will appear in a new window.

At the Select Columns to View popup, you can choose as few or as many fields as you need for your Cost Book. Once you have made your selections, click the Update button to save them, or click Cancel to discard them.

You can select the following fields from the Select Columns to View popup:

  • Work Action: Identifies and combines the Activity and Specification selected within the cost book.
  • Specification: The item selected for editing within the cost book.
  • Activity: The Global Repair, Replace or Demolish action required for the specified item.
  • Object Type: A generalized item classification compared to the Specification. For example, a Specification may be "Pipe, Copper, 8-12 Inch." The Object Type will simply be "Pipe."
  • Domain: The domain applied to the cost book, such as Utility-Gas, Waterfront, or Facility-Buildings.
  • Description: Text entered by the user to aid in identifying the cost book.
  • Level: The Asset or Component required for the specified item.
  • Parent: The parent Organization for the specified item.
  • Original Cost: The earliest value of the specified item.
  • Cost Year: The year used to calculate a specified item's inflation data.
  • UoM Unit of Measure: The Unit of Measure for the selected item(s).

(Note: the available UoM Unit of Measures depend upon the selected cost book and user requirements).

  • Minimum Cost: The base-level value of the cost book.
  • Materials: The value of materials used to create the cost book.
  • Labor: The value of labor used to execute the cost book.
  • Equipment: The value of equipment used for the cost book.
  • Bare Total: The estimated total value of the cost book.
  • Source: The basis of the cost book information, which is user-entered.
  • Frequency: Describes the rate at which work needs to be performed.
  • Crew: The number of people assigned to the cost book.
  • Daily Output: The cost book's output value per day.
  • Total Output: The cost book's overall value for the fiscal year.

You can select Filters by each field column () to narrow or expand your search. Results can be organized in ascending or descending order by selecting the minus icon by each field column. Click the minus icon once to get the Up caret, which outputs results in ascending order, click it twice to get the Down caret, which outputs results in descending order, or click it three times to get the minus icon again, which will revert the Cost Books to their original order.

If a Cost Book has been overridden or updated, you will see the Remove button (the trashcan icon) at the far right of the item row. Select this button to delete a Cost Book.

Editing a Cost Book

The following Cost Record fields cannot be edited: Work Action, Work Activity Type, Managed Object Type, and Specification.

Note: Any changes you make to the Cost Record screen will not be saved unless you click the Update button.

Note: You must Save your changes (via the floppy disk icon highlighted above) or your work will be lost if you select another Cost Book from the dropdown or navigate away from this page.
