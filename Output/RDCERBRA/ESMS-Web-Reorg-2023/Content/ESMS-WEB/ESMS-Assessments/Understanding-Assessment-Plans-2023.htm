Understanding Assessment Plans

This topic introduces tasks you can initiate at the assessment plans screen and provides links to greater detail. To reach the assessment plans screen, activate the Assessment Plans tile on the ESMS menu.

What Can I Do Here?

At the Assessment Plans screen, you can open an existing assessment plan listed in the table (see View Assessment Plans or Edit Assessment Plan), or (with appropriate permissions) create a new assessment plan (see Add Assessment Plans).

Note: Only assessment plans you have permission to view and/or work with will appear in the table.

Each type of assessment plan is described below, at About Creating An Assessment Plan .

Permissions Needed for Assessment Plans

  • To edit or view information in an assessment plan after it has been created, you need the ESMS permission named "Assessment Plan Editor." This can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission, to edit or view any assessment plan.
  • To create an assessment plan, you need "WorkplanRole" permission. This permission allows you to activate the Save button in the "Create Plan" popup. It can be set up either (a) as a scoped permission / role (restricted to a particular site), or (b) as a global permission.

These permissions are granted by an ESMS administrator at the Permissions tile, Users tab, Global Permissions and/or User Workspace section.

About Creating An Assessment Plan

Assessment Plan Setup

Before proceeding to the stages of the assessment plan, there is information to be defined when you create the assessment:

  • Name the assessment plan
  • Specify the site the assessment plan will be associated with
  • Select a domain
  • Set a budget
  • Set a planned assessment date
  • Assign users to PM, engineering, and/or reviewer roles

The setup will guide progress through the assessment plan, from resource planning to importing work items created in the field to including those work items into work packages.

If an appropriate plan is not available for selection, there are two alternatives:

  1. Edit an existing plan.
  2. Add a new plan. Alternatively, a brand new scheme can be added. You can design the sequential stages of creating an assessment plan using selected parameters.

Transitions

Transitions between the stages are governed by the user-designed scheme. Any actions necessary for transition to the next stage are shown in the tracking area at the bottom of the screen at each of the stages.

Scheme

A scheme is a definition of permissions and actions needed to transition from one stage to the next. The scheme customizes the assessment plan, and more than one scheme may be available to users.

When a user with appropriate permissions approves the transition and confirms it in the popup that appears when the Approve to Pre-Assessment Plan button is activated, then the assessment progresses to that stage.

Assessment Level Task

The following assessment plan tasks can be initiated at the Assessment Plans screen:

  • View Assessment Plans
  • Add Assessment Plans
  • Edit Assessment Plan
  • Delete Assessment Plan

Instructions for navigating to and in an assessment plan are in the topic Assessment Plan Navigation.

Stage Level Tasks

Other changes to an assessment plan can be made at the appropriate stage within the assessment plan. The stages of an assessment plan are determined by the scheme being used. Currently, ESMS consists of the following Workflow Stages:

  • Charter Stage
  • Pre-Assessment Tab
  • Review Stage
  • Post-Assessment Stage
  • Packages Tab
