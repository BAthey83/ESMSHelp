Enterprise SMS and SMS Inventory Differences

This topic is created to help with the transition from using legacy SMS to using Enterprise SMS. For a focus on differences in the inventory/asset tree between SMS and Enterprise SMS, see TURF THIS. Appendix A: Enterprise SMS vs SMS Inventory .

Topic Objectives

The objectives of this topic are:

  1. To highlight differences between SMS modules and Enterprise SMS when adding and working with assets.
  2. To introduce the new concept of groups, which exist as an organizational device apart from the asset tree.

Asset Tree Structure

For detail about the differences in structure between the Enterprise SMS asset tree and those of SMS modules, see TURF THIS. Appendix A: Enterprise SMS vs SMS Inventory . The following documentation sections describe some differences beyond the asset tree structure.

Adding Inventory

Differences in functionality and {objectives} between the Web and Desktop versions of Enterprise SMS lead to a "division of labor" for certain Enterprise SMS functions.

Inventory is added to Enterprise SMS only through the Desktop version of the software, not via the Web. After adding inventory, you can use the synchronize (Sync) tool in the Desktop to upload the inventory to the Web version of Enterprise SMS. In Enterprise SMS Web, Sync capability is limited to selecting the site to be synchronized.

Working with Inventory Items (Assets)

In Enterprise SMS, no asset functions (such as adding and deleting) are accomplished via right-clicking on the asset in the tree, as is the case in BUILDER and ROOFER. Functions permitted to the current user will appear on the toolbar at the Assets page.

Groups

In addition to the asset tree, groups are an optional, complementary device for organizing assets.

Using groups in Enterprise SMS is a way to associate assets together. "Group" is not an asset tree level, but rather an optional property of an asset. With the exception of the SMS Utilities module (which was developed in the same time frame as the beginnings of Enterprise SMS), groups are a new concept introduced by Enterprise SMS.

The selection of available groups is created by the end user's organization. A group may consist

  1. entirely of components;
  2. entirely of other groups; or
  3. of a mixture of one or more components and one or more groups.

Groups, as an organizational element, have two levels:

  1. Group-type
  2. Within each group-type are the groups.

Groups allow disparate assets to be clustered together according to whatever quality is desired, such as a common location or common purpose.
