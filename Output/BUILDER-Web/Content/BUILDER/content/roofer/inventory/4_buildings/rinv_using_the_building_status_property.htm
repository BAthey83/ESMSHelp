How to Use a Building's Current Status Property

The Current Status property allows you to maintain database records for Buildings that are not currently in your physical inventory, whether for historical purposes or future planning. A Building's Current Status is used in the decision processes of both ROOFER and BUILDER, and it is the basis for a number of useful capabilities, including allowing you to enter inventory for Buildings not yet built, or inventory and inspections for Buildings not yet acquired.

Set the Current Status Property

The Current Status property is set when you Add a Building. The default status in the "Add Building" popup is "Active."

Note: in the "Add Building" popup window, the Current Status property is labeled simply "Status."

You can change this default value either by using the dropdown list of status options at the time of adding the Building, or when you Edit Building General Information at the Building-level General Information screen.

Current Status can have any one of the following values:

  • Active. This is the default status. Use this status for Buildings that are currently in use. This status is the end status for Buildings with status "To be built" or "To be acquired."
  • Vacant. Use this status for Buildings that are currently in your physical inventory but not in use.

    Note: Currently there is no way to make ROOFER treat Buildings with this status differently from "Active" Buildings.

  • To be transferred. Use this status for Buildings that will be transferred to another owner in the future. The effective year of this status is the year in which the transfer will take place.

    Note: Currently there is no way to make ROOFER treat Buildings with this status differently from "Active" Buildings.

  • To be demolished. Use this status for Buildings that will be demolished in the future. The effective year of this status is the year in which the Building is to be demolished.

    Note: In the future it may be possible to assign lower prioritization to work items associated with a Building having this status, but currently there is no way to make ROOFER treat Buildings with this status differently from "Active" Buildings.

  • To be acquired. Use this status to enter inventory records for an existing Building prior to its acquisition. The effective year of this status is the year in which the Building is expected to be acquired.

    You will be able to edit the Building and inspection data for this type of Building, and ROOFER's condition and remaining service life predictions will be applied based on the inspection data entered. In this way, you will be able to estimate the maintenance requirements of such a Building prior to its acquisition. For more information, see "To Be Acquired" below.

  • To be built. Use this status to enter inventory records for a new Building prior to its commissioning for occupancy. The effective year of this status is the year in which the Building is expected to be recognized as part of your physical inventory.

    In ROOFER, you will be able to edit the Building data on the Building Summary screen but will not be able to add inspection records until the Current Status changes to "Active." For more information, see "To Be Built" below.

  • Transferred. Use this status to force ROOFER's automated processes for condition/service life assessment and work planning to skip the Building but leave the inventory and inspection records intact. The effective year for this status is the year in which the transfer takes place. "Transferred" the end status of the "To be transferred" status. For more information, see Keep Records of Past Buildings (Demolished, Transferred) below.
  • Demolished. Use this status to remove a Building from ROOFER's automated processes for condition/service life assessment and work planning but leave the inventory and inspection records intact. The effective year for this status is the year in which the Building is demolished. "Demolished" is the end status of the "To be demolished" status. For more information, see Keep Records of Past Buildings (Demolished, Transferred) below.

Review of Current Status Options:

The last four of these values indicate that the designated Building is not currently part of the inventory, and two others ("To be transferred" and "To be demolished") indicate that the Building will be removed from the inventory in the future. In essence, one of the things that the Current Status property allows you to do is to enter future Buildings into the inventory and to plan for their maintenance even though they are not presently owned. In addition, the Current Status property allows you to keep records for Buildings that are no longer in the physical inventory. Such records are ignored in processes that should only consider current Buildings, such as condition rollups and work planning.

Currently ROOFER treats the "Active" and "Vacant" status options in the same way.

The following sections provide tips on how current status property settings other than "Active" can be profitably used to avoid losing records and to track non-current inventory.

Use the Current Status Property to Avoid Losing Records

When you Delete a Building from ROOFER, this removes all database records associated with that Building. If you wish to keep the records, whether for historical purposes, analysis, or possible transfer to another ROOFER database (perhaps the Building is being transferred from one property book to another), you should simply set the Current Status property to an appropriate value instead, such as "Transferred" or "Demolished."

Add Future Buildings

Using the Current Status property, it is possible to enter Buildings into the database before they are actually in the physical inventory. This includes Buildings under construction (designated with status "To be built") and existing Buildings being considered for acquisition from another owner (designated with status "To be acquired").

"To Be Acquired"

By setting the Current Status property to "To be acquired" and entering data in advance, you will be able to anticipate short-term work requirements for existing Buildings about to be acquired.

Enter Inventory Records

Use this status to enter inventory records for a new Building prior to its acquisition.

Record Inspection Results

Once you have entered inventory records for a Building with status "To be acquired," you may record inspection results prior to acquisition. The ROOFER processes that determine remaining service life and condition prediction will track the condition of the Building's roof section after the Building is first entered and prior to its acquisition.

Generate a Work Plan

With inspection results, ROOFER will be able to generate a work plan for the Building to upgrade its roofing to your established standards. Such information could be invaluable in planning for the load such a Building will place on both short and long term budgets.

"To Be Built"

Enter Inventory Records

Use this status to enter inventory records for a new Building prior to its commissioning for occupancy.

No Recording of Inspection Results

If you enter a Building with status "To be built," you may not record new inspections until the status changes to "Active." The ROOFER processes that determine remaining service life and condition prediction will skip such a Building under the assumption that quality assurance during the construction process will force all roof sections to an RCI Roof Condition Index of 100 prior to commissioning.

Use Scenarios to Simulate "Active" Status

The advantage of entering the Building before it is built is that you may include it in Scenarios to measure its long-term effects on work plans and budgets That is, the scenario can change the status of a new Building to "Active" at a scheduled time and begin treating it as a normal Building with predictable deterioration rates and work requirements.

Keep Records of Past Buildings (Demolished, Transferred)

In earlier versions of ROOFER, the only way to eliminate a Building from the automated assessment and planning processes used by ROOFER was to delete the Building. With this approach, all records related to the Building were actually deleted from the database.

Now, the Current Status property gives you another option.

If you have records in the ROOFER database for a Building which is to be demolished or transferred to another owner, you can keep the inventory, inspection, and work history records in the ROOFER database, while at the same time making the Building invisible to ROOFERâ€™s automated processes, simply by changing the Building's status to "Demolished" or "Transferred."

Best Practice Recommendation: When you delete a Building, you actually remove all database records associated with that Building. If you wish to keep the records, whether for historical purposes, analysis, or possible transfer to another ROOFER database (perhaps the Building is being transferred from one property book to another), then you should simply set the Current Status property to the appropriate value, either "Demolished" or "Transferred."

Below is a summary of what happens when a Building has the status of either "Demolished" or "Transferred":

  • Inventory records are locked. You will be unable to change the Building's data properties or its composition.
  • Condition and Functionality Assessment records are locked. You will be unable to change existing records or to add new records.
  • Work Planning records are locked. You will be unable to change existing records or to add new records.
  • Cost data is not updated. The Building's cost data will not be affected by annual cost updates at the beginning of the calendar year.
  • The Building will not be considered during automatic work generation.
  • The Building will not appear in Scenarios simulations.
  • If the Building's status is changed to "Demolished" or "Transferred" during a Scenarios simulation, it will have records for the years prior to its status change and then after that date in the scenario, it will be treated as above. That is, there will be no further updates of data, no new inspections, no changes in condition or service life, etc.
