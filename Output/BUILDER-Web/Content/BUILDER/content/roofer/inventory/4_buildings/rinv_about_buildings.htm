About Buildings

A large amount of information is stored at the Building level of the asset tree. This topic describes what a Building is and how to use it in organizing roofing inventory. It also provides links to the operations that can be performed at the Building level.

Note: Because a Building record is almost always larger and more complex than any other hierarchical record, it will take longer for a Building record to process and appear on the screen than it will for records at other levels in the hierarchy.

The Building level is the lowest level in the SMS Sustainment Management SystemSustainment Management System hierarchy that is shared by both BUILDER and ROOFER. Below the Building level, ROOFER has just one more hierarchical level, the roof section.

Operations that can be performed on Buildings are:

  • Add a Building
  • Edit Building General Information
  • Edit Additional Building Inventory Data
  • Delete a Building

Information is also available on using the Current Status property of a Building. Available status options are Active; Demolished; to be acquired; to be built; to be demolished; to be transferred; Transferred; and Vacant.

  • How to Use a Building's Current Status Property
  • Use Current Status to Track Non-Current Inventory
