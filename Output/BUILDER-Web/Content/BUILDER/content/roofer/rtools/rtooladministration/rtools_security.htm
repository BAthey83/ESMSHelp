Password Policy

The Password Policy view in the Security window lists settings relating to user login to ROOFER; for example, number of bad logins allowed, maximum password length, etc. The Password Policy options allow the administrator to set password parameters for users.

Edit User Properties

To edit user properties:

  1. Select the name from the list of users.

    2. Note: If the list is long, you can narrow down the number of entries by using a filter as follows:

    1. Fill in either (1) all or part of the User Name in the blank field directly under UserName, or (2) all or part of the user's First Name or Last Name in the blank field directly under Last Name, First Name.
    2. Press the Enter key on your keyboard.
    3. If multiple names still appear in the list, click once on the desired user name.

  4. Activate Save on the toolbar.
  5. Activate Close on the toolbar. To see the full list of users, you might need to delete text that remains in the filter fields (see illustration below), then press the Enter key on your keyboard.

Administering Roles

Brief List of Roles

Once users have been created, they can be assigned to Roles. The Roles are allowed varying degrees of program access rights under the system. The different Roles in ROOFER include:

  • Administrators - can administer users, assign users to Roles, and change ROOFER application settings.
  • Inventoriors - can input and edit inventory information from the Inventory screen
  • Inspectors - have all the rights of Inventoriors, and can input and edit inspection information from the Condition Assessment screen
  • Inspection Supervisors - have all the rights of Inspectors, and can create inspection schedules
  • Work Planners - have all the rights of Inspection Supervisors, and can also create and edit work plans, prioritization schemes, and RSL Remaining Service LifeRemaining Service Life and cost books.
  • Master Planners - have all the rights of Work Planners, and can also edit any and all data saved by other users of the program. Scope of Role Assignment

In ROOFER, each user's Role privileges can be assigned and restricted to the appropriate hierarchical level (Organization, Site, Building, etc.). For example, a user can be assigned as a Work Planner either to the entire database globally, or, alternatively, to one or more Sites or Organizations only.

Note: Users are typically assigned their permissions Role at the Site level.
