Add a Work Item

The information stored with a ROOFER work item shows the following information:

  • What work was or is to be done.
  • When it was or is to be done.
  • Its cost.
  • Its funding source.
  • Its importance relative to the other work items vying for the same resources at the same time.

In ROOFER, roof section-level work items can be created two ways, each of which is discussed in detail further below.

  • Manual Input - This option allows you to create individual work items manually from the work plan screen.
  • Generated Work Items - This option allows you to use the automated work generation feature in ROOFER to create an entire work plan.

Create a Work Item Manually

To create a work item manually,

  1. Activate the Add button at the Work Plan A multi-year collection of work items and/or projects planned for a Site or Organization. Items tab.

  1. Select the Building Work Item A discrete work activity applied to a single managed object such as a Component-Section, intended to improve its life cycle performance. radio button option in the "New Work Item A discrete work activity applied to a single managed object such as a Component-Section, intended to improve its life cycle performance." popup window that appears. A dropdown list will appear.

  1. From the dropdown that appears after you make your radio button selection, select the Building associated with the work item(s) to be entered.
  2. Activate the Proceed button on the popup toolbar.

The "Building Work Item A discrete work activity applied to a single managed object such as a Component-Section, intended to improve its life cycle performance." popup window will appear:

On this screen, the following data for the Section-level work item is displayed and can be edited. The descriptions below start with items in the left-hand column, then proceed to describing items in the right-hand column.

  • Description (Read-Only). Displays the description Component-Section of the work item.
  • Work Request ID. Enter your agency's work request number. Limit: 50 alphanumeric characters. This item of data is not used by ROOFER in any analyses.
  • Funding FY. Enter the four-digit fiscal year that the work item was or will be funded.
  • Work FY. Enter the four-digit fiscal year that the work item was or will be completed.
  • Date Completed. Enter the date the work item was completed. (Appears only if work has been completed.)
  • Score (Read Only). If you have prioritized the work items, then this work item's score will be displayed here. The default value is 0.000, indicating that the work items have not been prioritized.
  • Work Code. Select the work code of the work item. The available work codes are:
    • Sustainment
    • Modernization
    • Demolition
    • New Footprint
  • Work Activity. Select the work activity associated with this work item from the dropdown list. The work activities are:
    • Add/Install
    • Alter
    • Paint
    • Remove
    • Repair
    • Replace
  • Status. Select the current status of the work item from the dropdown list of choices. The status will change over time as the work item moves from planning to completion. The values of status are:
    • Awaiting Contract Award
    • Awaiting Funds
    • Awaiting Funds (Special Project)
    • Budgeted
    • Canceled
    • Completed
    • Deferred
    • Funded
    • Funded (Service Call)
    • Funded (Special Project)
    • In Design
    • In Progress
    • On hold
  • Must Complete as Planned. Check this box if the work item falls into the "must do" category. Marking this checkbox will force the work item to be funded before all other competing work items that are not marked as "must do" in the Scenarios module.
  • Funding Source. Select the funding source to be used for this work item from the created fund sources.
  • Estimated Cost. Enter the estimated cost of the work item. ROOFER provides a means of estimating the initial cost once you have entered the Fund FY, Activity, Cost Book, and Quantity using the Estimate button. Note that the cost module is unable to estimate the cost of activities to Add/Install and Alter, so the Estimate button is grayed out for those work activities.
  • Override Automatic Cost Estimate. Mark this checkbox if you wish to override the cost estimated using the Calculate button and enter your own estimated cost of the work item.
  • Actual Cost. Enter the actual cost of the work item after completion.
  • Cost Book. Select the cost book to be used from the dropdown list.

After entering the data for the work item, activate the Save button in the popup window toolbar to save the work item, then activate Close when you are ready to return to the Work Plan A multi-year collection of work items and/or projects planned for a Site or Organization. Items tab.

Alternatively, if you do not wish to save the work item, or the most recent changes made to it, just activate the Close button on the popup window toolbar.

WARNING: If you close before saving, you will lose your work.

At the Work Plan A multi-year collection of work items and/or projects planned for a Site or Organization. Items tab, your new work item(s) will appear in italic type, and have the Status of "Awaiting Funds."

Generating Work Items in ROOFER Automatically

As an alternative to manual generation of work plan items, you may use the Generate Items button to create a work plan for the current year.

After activating the Generate Items button, the Generate Work Items window will appear.

Activate Proceed to generate the work items.

WARNING: Activating the Proceed button will cause all of the currently existing work items in the work plan for the current fiscal year to be deleted.

ROOFER will determine where and what type of work should be done. Additionally, the estimated cost of each work item is computed using the cost book assigned to the Site in the Site General Information tab.. When the process is complete, the work plan will appear in the Work Plan A multi-year collection of work items and/or projects planned for a Site or Organization. Items window, displaying all of the newly generated work items. All of the newly generated work items will have the status "Awaiting Funds." You may then refine the work item list as you wish.

Note that ROOFER's work generation algorithms are still primitive in the sense that they are unable to detect work conflicts and precedences. For example, they may generate a need to repair a roof deck without a concurrent effort to repair the roof surface. Your own manual processing of the work item list must address such problems.

Export the Work Plan to Excel

The current work plan can be exported to a Microsoft Excel spreadsheet using the Export button. After activating this button, you must choose either to open or to save the Excel spreadsheet. If you choose to save, you must specify the location and name to save the exported spreadsheet.
