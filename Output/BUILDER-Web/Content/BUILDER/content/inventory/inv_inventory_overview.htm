Inventory Overview

This topic provides a brief overview of the inventory process, both for those approaching the task of planning and implementing an initial inventory of assets, and for those who will be updating and managing an existing real property inventory database in BUILDER™.

Those planning an initial inventory should also read Planning Inventory.

New BUILDER Implementation

BUILDER inventory serves as the foundation for all that your organization will do with the BUILDER application. The first step in a new implementation of BUILDER is to construct a database that describes the physical inventory that is to be managed. That process of putting the data into BUILDER will probably be the single most intensive effort in working with the program, but it is a one-time effort that can be aided by features that allow you to import data from other Systems; to copy Buildings; and to use Building templates when multiple Buildings have a single or similar design.

Depending on manpower and other considerations, your organization may choose to perform inventory and initial inspections (condition assessment) for a new BUILDER implementation at the same time. Alternatively, inventory can be performed first, and then inspections.

Initial inventory demands significant resources. Some organizations choose to hire either contractors or additional staff temporarily for the task of populating the inventory tree. However, the task of planning the scope and structure of the asset tree is typically best done by someone already familiar with the relevant organization.

The topic Planning Inventory provides advice for planning the structure of your inventory and for determining how to divide inventory into Systems, Components, and Sections.

Managing and Updating Inventory

After a description of physical property has been entered into the database during initial inventory–or possibly in parallel to initial inventory, if the inventory tree is being filled in gradually–comes the task of managing and updating inventory. Representative tasks of this phase are:

  • Newly acquired Sections, Systems, or Buildings need to be added to the inventory tree.
  • As Components and Component-Sections are replaced due to poor condition or modernization, the corresponding records need to be updated in the database.
