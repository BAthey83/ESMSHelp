Perform a First Tier Functionality Assessment

This topic covers how to perform a first tier functionality assessment using BUILDER. Functional assessments can also be performed using BuilderRED.

A first tier functionality assessment is associated with a Building as a whole. Default functionality assessments provided with non-customized instances of BUILDER are the Functionality assessment set, the Cultural Resources assessment set, and the HPSB High Performance and Sustainable BuildingHigh Performance and Sustainable Building assessment set.

Prerequisites

The following are required before performing a first tier functionality assessment:

  1. Create the Building. This needs to be done in Inventory mode in BUILDER or BuilderRED. If you have not created the Building in Inventory mode with proper data down to the Component-Section level, you will not be able to enter functionality assessment data for it.
  2. Be familiar with the concepts in the Functionality Assessment Overview and in First Tier Functionality Issues and Sub-Issues.

Note: With the addition of the Current Status property in BUILDER , it is possible to have functionality assessment records for non-current Buildings. Performing assessments on non-current Buildings is done in the same manner as performing them on current Buildings.

The following sections tell you how to create a first tier functionality assessment in BUILDER. and how to complete an assessment.

First Tier (Building) Functionality Assessment in BUILDER

Create or Open

When you are ready to add or open a functionality assessment data for a Building,

  1. Activate the Functionality ribbon in the Navigation pane (it may be semi-hidden at the bottom of your screen).
  2. Navigate to and select the desired Building/Facility.
  3. Verify that you are at the Assessments tab in the content area of the screen. Any existing assessments will show in a table in the content area.

  4. To open an existing assessment, activate the assessment's row in the table. You can now skip to Fill In the Assessment.
  5. To start a new assessment, activate the button at the upper left of the table. The "New Assessment" popup will appear.
  6. Select the desired assessment type by activating the radio button next to it.

  7. Activate the button at the bottom of the popup. You will be brought to a new instance of the selected assessment type.
  8. For now, activate Save, located at the upper left of the content pane, so that the assessment will be saved in the table of assessments even if it is not yet complete.

After performing the steps above, you should be inside the selected assessment.

To complete the assessment, use the radio buttons or checkboxes to answer each question.

Saving your work periodically as you work through the issues and sub-issues is a good idea.

Rating Sub-Issues with Wizards

For some sub-issues in the Functionality assessment, a wizard is provided to help you perform the rating for that sub-issue. To launch a wizard, activate the Assessment Wizard button included in the sub-issue row and follow the instructions in the wizard.

Adding or Editing Comments

By activating the Comment icon, you can create or edit comments for most of the sub-issue questions in the functionality assessment. A checkmark on the Comment icon indicates the presence of a comment.

Adding Images to the Assessment

Wherever you see the Images icon, you can attach an image that supports understanding of the sub-issue and/or its assessment. A checkmark on this icon indicates that one or more images are already associated with the assessment.

Note: The image you attach must be in one of the following formats: .png, .bmp, or .jpg/.jpeg

To attach an image,

  1. Activate the Images icon.
  2. In the "Photo" popup, activate the Choose File button.
  3. Browse to the location(s) where you have images you want to attach to the assessment.
  4. Select the desired image. BUILDER will insert the image and close the file browser window.
  5. Repeat steps 2 to 4 as desired.
  6. When finished adding image attachments, activate the Save button. Alternatively, activate the "X" in the upper right corner of the popup if you don't want to add the selected image attachments.

Save Assessment

When you are finished filling in the functionality assessment, activate Save at the upper left of the content area, then Close.

  • Result (Read-Only). This is the functionality index of the assessment. Before assessment questions are answered, this will be 100 .
  • Delete. The Delete button gives the user the option to delete past assessments.

    • Note: Deleting a functionality assessment is not allowed in BuilderRED.

To inspect further data, choose which functionality assessment you wish to see data for, and activate its row in the table. All functionality assessments that have been previously recorded (and not deleted) will be accessible from this table.

Once you open a functionality assessment, the following data will be displayed at the top of the assessment:

  • Date.
  • Status (Current or Future).
  • Use Type.
