Perform a Second Tier Functionality Assessment

This topic covers how to perform a second tier functionality assessment using BUILDER. Functional assessments can also be performed using BuilderRED.

Second tier functionality assessments are performed on individual Functional Areas in the Building. Default functionality assessments provided with non-customized instances of BUILDER are the Functionality assessment set and the Cultural Resources assessment set.

Prerequisites

The following are required before performing a second tier functionality assessment:

  1. You must first create Functional Areas in the Building with the proper data. If you have not done so, you will not be able to enter this level of functionality assessment data.
  2. Additionally, you should be familiar with the concepts in the Functionality Assessment Overview and the Second Tier Functionality Issues and Sub-Issues.

Second Tier Functionality Assessment in BUILDER

Create or Open

When you are ready to add or edit functionality assessment data for a Functional Area,

  1. Select the Functionality ribbon in the left pane of the BUILDER application (the ribbon may be either at the top or the bottom of the left pane). The functionality assessment navigation tree will appear.
  2. Navigate the tree to the Functional Area you wish to perform the functionality assessment on.
  3. Select the Functional Area. (see "Lab" in the screenshot below)
  4. Activate the Assessments tab in the content area.

  5. To open an existing assessment, activate the assessment's row in the table. You can now skip to Select Components.
  6. Activate the button (below the Components tab).

    Note: If you do not see the New button, it may be because you only have read privileges for this location.

  1. In the "New Assessment" popup window, view the available options. The baseline assessment set is called Functionality. At the Functional Area level, there is also an assessment set for Cultural Resources.

  1. For now, activate Save, located at the upper left of the content pane, so that the assessment will be saved in the table of assessments even if it is not yet complete.

If you want to see more detail about assessment metadata, see Functionality Assessment Data Overview below.

Select Components

In a new Functional Area-level functionality assessment, you first need to select which components to include in the assessment (by System, or you can select at a lower level if desired). Use the checkboxes shown to select by System; or, activate the arrows to expand the inventory items shown, to lower levels.

 

Next, follow instructions given in Fill In the Functionality Assessment.

Fill In the Functionality Assessment

If an assessment is not already open, you can open one by selecting it from the table of functionality assessments shown at the Assessments tab:

To complete the assessment, use the radio buttons or checkboxes to answer each question.

Saving your work periodically as you work through the issues and sub-issues is a good idea.

Rating Sub-Issues with Wizards

For some sub-issues in the Functionality assessment, a wizard is provided to help you perform the rating for that sub-issue. To launch a wizard, activate the Assessment Wizard button included in the sub-issue row and follow the instructions in the wizard.

Adding or Editing Comments

By activating the Comment icon, you can create or edit comments for most of the sub-issue questions in the functionality assessment. A checkmark on the Comment icon indicates the presence of a comment.

Adding Images to the Assessment

Wherever you see the Images icon, you can attach an image that supports understanding of the sub-issue and/or its assessment. A checkmark on this icon indicates that one or more images are already associated with the assessment.

Note: The image you attach must be in one of the following formats: .png, .bmp, or .jpg/.jpeg

To attach an image,

  1. Activate the Images icon.
  2. In the "Photo" popup, activate the Choose File button.
  3. Browse to the location(s) where you have images you want to attach to the assessment.
  4. Select the desired image. BUILDER will insert the image and close the file browser window.
  5. Repeat steps 2 to 4 as desired.
  6. When finished adding image attachments, activate the Save button. Alternatively, activate the "X" in the upper right corner of the popup if you don't want to add the selected image attachments.

Save Assessment

When you are finished filling in the functionality assessment, activate Save at the upper left of the content area, then Close.

Functionality Assessment Data Overview

After a Functional Area is selected, the general data that was input when creating the Functional Area will be shown at the top of the screen:

  • Name (Optional) . The name or a brief description of the Functional Area.
  • Area. Size in the selected units of measurement (SF or SM)
  • Area Type (Functional Use) (Required). Select the functional use from the dropdown list that most closely matches the Functional Area. BUILDER allows you to choose from 26 different Functional Area types. This field is helpful when a Functional Area is being assessed against many functional uses to determine its optimal functional use.
  • Use Type (Required). From the dropdown list, select the use type that most closely matches the Functional Area. If you are a Department of Defense activity, BUILDER should display familiar category codes for your service. This field is helpful when a Functional Area is being assessed against many use types to determine its optimal use type.

Additionally, to the right of this general data two indexes are displayed:

  • FI Functionality IndexFunctionality Index (Assessment FAFI Functional Area Functionality IndexFunctional Area Functionality Index) (Read-Only). Displays the FAFI Functional Area Functionality IndexFunctional Area Functionality Index, computed based on the data recorded for the assessment currently selected.
  • CI Condition IndexCondition Index (Read-Only). The condition index of the items included in the Functional Area.

To inspect further data, activate the Assessments tab. Then select from the table of assessments the functionality assessment you wish to see data for by activating its row in the table. All functionality assessments that have been previously recorded (and not deleted) will be accessible from this table. For each of them, the inspector name and date of the inspection is provided. If you have created a new functionality assessment, the current date will be shown in the "Date" field.

Inside the functionality assessment, the following data are in the header above the list of assessment issues:

  • Status (Required). Select the status of the current assessment from the dropdown list. The status options available for a functionality assessment in BUILDER 3.6 are:
    • Current. Sets the current functionality assessment to active and is used to compute the current FAFI Functional Area Functionality IndexFunctional Area Functionality Index.
    • Future. Sets the current functionality assessment to a future year when user requirements, codes, or obsolescence are expected to change and affect the functionality of the area. These assessments are particularly useful to use when running Scenarios.
  • Effective Date (Conditional). Enter the year the status of the assessment will become active, or effective in the Functional Area. This field will be operational only if the Status is set to Future. Enter the date in the field provided, or else use the calendar to select a date from the calendar.
  • Use Type (Required). Select the use type from the dropdown list that most closely matches the Functional Area. If you are a Department of Defense activity, BUILDER should display familiar category codes for your service. This field is helpful when a Functional Area is being assessed against many use types to determine its optimal use type.
  • Date (Required). Shows the date the assessment was created. Changes may be made to the assessment only within a 30-day window from this date.
