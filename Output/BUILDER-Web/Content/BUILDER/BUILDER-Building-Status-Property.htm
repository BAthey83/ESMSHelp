About the Building Current Status Property

The Current Status property of a Building allows you to maintain database records for Buildings not currently in your physical inventory, whether for historical purposes or future planning. A Building's current status is used in the decision processes of both BUILDER and Scenarios, and it is the basis for a number of useful capabilities, including allowing you to:

  • Define policies so that Buildings scheduled for demolition are maintained at a lower standards than might generally apply. Simply use the Current Status as one of the attributes used to define a policy and assign standards as appropriate to each status type.
  • Define work prioritization schemes that consider future changes in a Building's status in prioritizing work items. This can be done by using the Current Status property as an initial split in your prioritization scheme and assigning weights appropriate for each status type (using 0 weight for non-current status types).
  • Plan for the maintenance load of Buildings to be added to your inventory in the future by analyzing a Scenario.
  • Change the status of a Building dynamically during a Scenario simulation to study the effects of a changing inventory on the budget and the work load.

Note: Select Building-level field values have been locked for general users in order to maintain the integrity of data and systems connections. For additional information, please contact your data manager.

The Current Status property is set on the Building data screen at the General Information tab (see Edit Building General Information). Local variation (which is typically in the form of an added option) is possible due to database customization, but generally the Current Status property has one of the following values:

  • Active. This is the default status. Use this status for Buildings that are currently utilized for 6 months or more annually. This status is the end status for Buildings with the status "To Be Acquired."
    • Note: This status generates work items.
  • Caretaker. Select this status when a Building requires minimal maintenance for safety and security.
    • Note: This status generates work items.
  • Closed.Choose the Closed status when a Building is not currently being used or maintained.
    • Note: This status does not generate work items.
  • Disposed. Use this status when a Building has been demolishedâ€”or when a Building has been conveyed or transferred out of the government's inventory.
    • Note: This status does not generate work items.
  • Environmental Hold. Select Environmental Hold when a Building is not currently in use due to any environmental issues or concerns.
    • Note: This status does not generate work items.
  • Excess. Use the Excess classification when a Building is no longer required to meet current or future needs of the Federal Government, but it has been approved for future federal use.
    • Note: This status generates work items.
  • Non-Functional. Choose the Non-Functional option when a Building cannot be occupied or used for any purpose until functionality returns to a habitable or usable condition.
    • Note: This status generates work items.
  • Outgranted. Select the Outgranted status when Building rights have been granted to a non-DoD agency.
    • Note: This status does not generate work items.
  • Semi-Active. Choose this status if the Building is used less than 6 months a year (such as training centers).
    • Note: This status generates work items.
  • Surplus. A Building can be classified as a Surplus asset when it is not required to meet current or future federal governmental needs and responsibilities.
    • Note: This status does not generate work items.
  • To Be Acquired. Use this status to enter inventory records for an existing Building prior to its acquisition. The effective year of this status is the year in which the Building is expected to be acquired. You will be able to edit the Building and inspection data for this type of Building, and BUILDER's condition and remaining service life predictions will be applied based on the inspection data entered. In this way, you will be able to estimate the maintenance requirements of such a Building prior to its acquisition. In a Scenarios simulation, this feature allows you to acquire Buildings as the simulation progresses.
    • Note: This status does not generate work items.

Two of these values indicate that the designated Building is not currently part of the inventory: To Be Acquired and Disposed.

The Current Status property allows you to enter future Buildings into the inventory and to plan for their maintenance in Scenarios even though they are not presently owned. In addition, the Current Status property allows you to keep records for Buildings that are no longer in the physical inventory. Such records are ignored in processes that should only consider current Buildings, such as condition rollups and work planning.

Best Practice Recommendation: When you delete a Building, you actually remove all database records associated with that Building. If you wish to keep the records, whether for historical purposes, analysis, or possible transfer to another BUILDER database (perhaps the Building is being transferred from one property book to another), you should simply set the Current Status property to an appropriate value.
