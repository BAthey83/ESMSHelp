Managing Users and Roles (Security Screen)

This topic introduces tasks you can perform at the Security screen.

Note : The Security screen is viewable by ESMS administrators only. It will not appear on the Dashboard of a non-administrator.

What Can I Do Here?

At the Security screen you can use the Security Editor to add and delete users, create user roles with custom security permissions, and assign roles to users.

To view or edit user details, locate the username in Security Editor or search for the username by typing the name in the Search bar. A right arrow next to the user's name in the Users list indicates the user is a Parent user.

Users

At the Users tab in the Security Editor, you can add or delete a user, and view or edit a user's details and roles/permissions.

To add or edit a named role, see Roleson page 1.

View User Details

To view user details,

  1. At the Security Editor screen, select the Users tab. A list of usernames should appear in the left pane.

    Note: A right arrow next to the user's name indicates a Parent user. Activate the arrow to see child users.

  2. To search for a user, enter all or a portion of the username in the search bar above the names.
  3. Select the username in the list of users. User details will appear in the right pane.
  4. See the topic User Detailson page 1 for an overview of user details.

Add User

To add a user,

  1. Activate the "+" icon on the footer toolbar. The "Create User" popup window will appear.
  2. In the "Create User" popup, select a Parent user if you want the new user to be a "child" of a top-level user. To create a top-level user, leave as "None Selected".
  3. Next, specify username, password, and email in the popup.
  4. Activate the Create button. Alternatively, you can Cancel. You will be returned to the Security Editor screen.
  5. Enter user information at User Details. See the topic User Detailson page 1 for an overview of user details.
  6. Alternatively, to close the window without creating the User, activate the Cancel button.
  7. Additional user details are described in the topic User Detailson page 1.

Edit User Details

To edit user details,

  1. At the Security Editor screen, select the Users tab. A list of usernames should appear in the left pane.

Note: A right arrow next to the user's name indicates a Parent user. Activate the arrow to see child users.

  1. Locate the username in the user list or search for the username by typing in the search bar.
  2. Select the username in the left pane. User details will display in the right pane. See the topic User Detailson page 1 for an overview of user details.
  3. Change fields as required to edit user information.
  4. To save changes, activate the Save button at the bottom of the User Details page.
  5. Alternatively, to close the User Details windows without saving changes, activate the Cancel button.

Assign or Edit User's Permissions / Roles

To assign or edit a user's permissions / roles, see Assign Role to Useron page 1.

Delete User

To delete a user,

  1. Select the username in the user list at the Users tab. (To search for a user, enter all or a portion of the username in the Search bar.)
  2. If you want to delete a Parent user, delete the children first.
  3. With the username selected, activate the "X" icon on the footer toolbar.
  4. Activate Delete in the confirmation popup to delete the user. Alternatively, activate Cancel to cancel the operation.

Assign Role to User

Two types of roles (permissions) can be assigned to a user: Scoped Permission / Role ("User Workspace" Section)on page 1, or Global Permission / Roleon page 1.

Scoped Permission / Role ("User Workspace" Section)

The "User Workspace" section of User Details is where you assign a scoped role to a user. At the Organizations tab in User Workspace, you can add or delete a user's role to a specific organization. At the Sites tab in User Workspace, you can restrict the user's role to a specific site. At the Groups tab in User Workspace, you can assign a role to a user group as a whole.

To assign a role to a user,

  1. To select the scope for the role (site vs. group), select either the Sites tab or the Groups tab in the user workspace.
  2. Activate the Add Site Role or Add Group Role button, as applicable.
  3. In the "Add Site/Group Role" popup, activate the None Selected button at the Site or Group field, then select the desired site or group using the new popup that appears.
  4. Next, select a role from the dropdown in the Role field.
  5. Activate Create at the bottom of the "Add Site/Group Role" popup to assign the role to the selected user. Alternatively, use the Cancel option provided.

Global Permission / Role

Assign global permissions in the Global Permissions section.

For example,

  • To edit or view information in stages of any assessment after it has been created, the user needs "Assessment Editor" global permission.
  • To create a assessment, one either with or without workflow, the user needs "AssessmentRole" permission. (This permission allows the user to activate the Save button in the "Create Assessment" popup.)
  • To see the Workflow Editor tile on the Dashboard and to edit or create schemes that govern the workflow through an assessment, the user needs "WorkflowRole" permission.

Note: A non-administrator user's "Assessment Editor" or "AssessmentRole" permission can be restricted to a specific site by using the User Workspace section instead of Global Permissions. See Scoped Permission / Role ("User Workspace" Section)on page 1. The "WorkflowRole" permission is global only.

CAUTION: Because the workflow scheme design process is very open and flexible, "WorkflowRole" permission should be given out sparingly, to those who can anticipate the consequences of their design decisions.

Roles

Add Role

To add a role,

  1. At the Team Editor screen or (for ESMS administrators) the Security Editor screen, select the Roles tab.
  2. Make sure that you do not have any of the existing roles selected. (For what happens when a role is selected when you activate the "+" button in the toolbar, see Managing Users and Roles (Security Screen)on page 1 "Copy Existing Role" below.)
  3. Activate the "+" button at the top of the screen.
  1. In the "Create Role" popup, enter a name for the new role in the "Role Name" field. Spaces are allowed.
  2. To confirm, activate the Create button.
  3. Alternatively, activate the Cancel button to cancel adding the role.
  4. To specify permissions for the role, first activate the role in the list of roles. The role name and a list of possible permissions will appear under "Select Role".
  1. Select the desired combination of permissions by clicking each appropriate checkbox.
  2. Activate the Save button. A confirmation message should appear.

Copy Role

You can create a new copy of an existing role, which will be created as a subsidiary of the role copied. You can then alter the permissions in the subsidiary as desired.

To copy a role into a subsidiary of that role:

  1. At the Team Editor screen or (for ESMS administrators) the Security Editor screen, select the Roles tab.
  2. Select the role you want to copy. It should become highlighted to show your selection of it.
  3. While the role shows as selected, activate the "+" button in the toolbar at the bottom of the screen. A popup will appear.
  4. Type a name for the copy in the "Name" field.
  1. Activate the Create button. The copy will appear in the list of roles, indented under the role it was copied from.
  2. IMPORTANT: select the copy in the list of roles. Otherwise you will be editing the role it was copied from, despite that the copy's name appears in the "Name" field.
  3. Edit the permissions of the copy as desired.
  4. Activate the Save button.
