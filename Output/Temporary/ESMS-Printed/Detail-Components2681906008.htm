Component Tasks

Most tasks relating to components–such as adding, changing properties, and deleting–are done in Enterprise SMS Desktop. However, a component task that can be performed in Enterprise SMS Web is to

  • upload a document to be associated with the component (this task is described in the topic Associate Document(s) with a Component)

Navigating the Component Details Screen

When you select a row in the results table at the Component tab of the Search screen (a single click suffices), the detail screen for that component will appear. For example:

Component Properties

At the Component Details screen, you can see the following domain-agnostic properties for the selected component, if the properties have been entered into Enterprise SMS Web:

  • component name
  • component type
  • parent site
  • facility number
  • quantity
  • RPUID Real Property Unique Identifier. This is a facility-level property, described in JIRB 2004-07-19, Department of Defense Real Property Inventory (RPI) Initiative, dated July 19, 2004.
  • date installed
  • operational status
  • UNIFORMAT classification
  • MDI Mission Dependency Index
  • replacement value
  • replacement cost
  • specification
  • number
  • repair date
  • alternate ID
  • CI Condition Index
  • FI Functionality Index.
  • FCI Facility Condition Index.
  • EFCI
  • PI Performance Index. The Performance Index is a weighted combination of the CI value and the FI value, with the lower of the two values being given two-thirds of the weight and the higher of the two values being given one-third of the weight. If no functional assessment has been performed, the FI is assigned a value of 100.
Accordion Button Options

At the bottom of the screen you can activate any of the accordion buttons to toggle the section open or closed. An explanation of information available at each button follows.

Custom Attributes

Shows attributes (properties customized to the specific domain) associated with the current component.

Assessments

Shows assessments associated with the current component.

Work Items

Shows work items associated with the current component.

Inspection Schedule

Shows the inspection schedule established for the current component.

Child Components

Shows any child components that may be in the current components.

Documents

Shows documents (images, diagrams, etc.) associated with the current component. Here you can also upload additional documents to be associated with the component (see Associate Document(s) with a Component) .

Cost Modifiers

Shows the cost modifiers associated with the current component.

Performance Records

Shows the performance records associated with the current component.
