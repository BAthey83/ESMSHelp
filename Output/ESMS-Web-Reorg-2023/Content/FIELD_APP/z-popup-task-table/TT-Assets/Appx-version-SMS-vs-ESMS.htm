TURF THIS. Appendix A: Enterprise SMS vs SMS Inventory

This topic is created to help with the transition from using legacy SMS to using Enterprise SMS. For other differences between SMS and Enterprise SMS, see Enterprise SMS and SMS Inventory Differences.

Topic Objectives

The objectives of this topic are:

  1. To show the structure used in Enterprise SMS to unify the asset tree structure between all of the Enterprise SMS domains (formerly separate modules in SMS).
  2. To introduce the new concept of groups, which exist as an organizational device apart from the asset tree.

Unified Inventory Structure

For the first time, information from the separate modules of the Sustainment Management System (SMS) can be viewed together, in Enterprise SMS. Each of these modules may have had a different asset tree structure below the Site level than is now provided in Enterprise SMS. It is the purpose of this appendix to provide a reference to the asset tree structure and nomenclature of each of the modules, and to outline how each structure maps to the Enterprise SMS structure of facilities, groups, and components. This comparison is available from two angles:

  • Pairwise comparisons between Enterprise SMS and each of the SMS modules
  • Comparison of each Enterprise SMS asset tree level with all SMS modules

Groups

Using groups in Enterprise SMS is a way to associate assets together. "Group" is not an asset tree level, but rather an optional property of an asset. With the exception of the SMS Utilities module (which was developed in the same time frame as the beginnings of Enterprise SMS), groups are a new concept introduced by Enterprise SMS.

The selection of available groups is created by the end user's organization. A group may consist

  1. entirely of components;
  2. entirely of other groups; or
  3. of a mixture of one or more components and one or more groups.

Groups, as an organizational element, have two levels:

  1. Group-type
  2. Within each group-type are the groups.

Groups allow disparate assets to be clustered together according to whatever quality is desired, such as a common location or common purpose.

Summary

Enterprise SMS unifies the inventory structure between all of its domains: Buildings, Fuels, Pavements, Rails, Roofs, and Utilities. The amount of difference you will see in the inventory tree structure and nomenclature depends on which SMS program you have been using.

An additional, new organizational device introduces in Enterprise SMS is the group. Groups (which exist as a property of a component or of another group) allow disparate assets to be clustered together according to whatever quality is desired, such as a common location or common purpose.
